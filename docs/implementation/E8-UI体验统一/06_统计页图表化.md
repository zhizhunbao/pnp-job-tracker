## 1. 整体目标

/stats 省索引(页面版与 /jobs 统计弹窗共用同一 `StatsIndexContent`)从纯卡片/表格升级为**图表优先**:顶部一组「常见图表」预设(在招职位/中位年薪/清单岗 按省,全国大类岗量),下方「自定义图表」= 维度(按省 / 按职业大类)× 指标(6 个)× 可选过滤(某大类 / 某省)自由组合。**零新数据、零新接口**——stats 表 119 行(10 省 × [all + 10 大类 + 未分类])已含全部组合,页面/弹窗数据链路不变,只改内容组件。

## 2. 验收标准

- [ ] /stats 与统计弹窗顶部出现 4 张常见图(按省在招 / 按省中位年薪 ESDC / 按省省提名清单岗 / 全国各大类在招),条形图降序、悬浮 tooltip、三语标签。
- [ ] 自定义区:维度切换(省/大类)、指标 6 选 1、过滤下拉(维度=省时可锁某大类;维度=大类时可锁某省),任意组合即时出图。
- [ ] **数据诚实红线**:中位数指标在「维度=大类 × 全部省」时**不做跨省合并**——显示三语提示引导选省,绝不显示中位数的中位数。
- [ ] 未分类不进大类维度图(与省页表格口径一致);计数类指标跨省求和,中位类只透传 ETL 预聚合值。
- [ ] echarts 按需懒加载(动态 import,仅图表挂载时拉),/jobs 首屏 bundle 不增重;两端(1440/iPhone13)图表可读、无横向溢出。
- [ ] 省卡片导航/跨省对比入口/口径行保留(图表区之下),弹窗端 state 导航不受影响。

## 5. 现有代码 / 决策记录

- **选型 ECharts**(vs Recharts/victory/nivo):决策阶梯走到第 4 级(现装依赖无图表库)→ 用户点名「开源的漂亮图表」;ECharts 默认审美/tooltip/动画开箱即漂亮,dataset+encode 天然适配「任意维度×指标」;体积劣势(full ~340KB gz)用**动态 import 懒加载**抵消——只有打开统计页/弹窗才拉。不引 `echarts-for-react` wrapper(30 行 useRef 壳自己写,少一个依赖)。
- **中位数红线**:stats 表没有「全国×大类」预聚合行,中位数的跨省合并在数学上不成立(中位数的中位数≠合并中位数)——宁可提示引导,不瞎算(CLAUDE.md 清洗红线在展示层的延伸)。
- 计数类(在招/近7天/清单岗/AIP)跨省求和成立,全国大类图用它。
- 图表放 `StatsIndexContent` 内=页面与弹窗自动同款(E8-02 内容组件单一来源原则);省页/大类页本轮不动(表格信息密度已够,后续按用户反馈加)。
- 条形图用横向(category 在 y 轴):10 个省/大类名在手机窄屏竖排也可读,竖向柱状图的 x 轴标签会挤成斜排。
