# E8-10 弹框三合一收编(24 → 3)

> 状态:**设计待实施**(2026-07-21 Frank 拍板方向,清单待他逐项过)
> 前置:`docs/弹框规范.md`(壳+骨架,2026-07-05)、`docs/顾问弹框字段规范.md`(五段骨架+九族,D1-D4 已拍板未实施)

## 1. 为什么做

Frank(2026-07-21):「公司的话应该只描述公司的信息,职位的话只描述职位相关的信息…Indeed 没有拆分这么细,我们是按字段弹框。让我减少弹框,比如地点这种就不谈了。」

现状:**34 个 ColKey,24 个能点开顾问弹框**,一套 `AdvisorModal` 伺候全部字段。代价:

- **一半以上是一句话**。`SIMPLE` 集合 13 个字段(见 `advisor/route.ts:155`)明确标着「一句话,不分段」——为一句话开弹框,用户要点一下、等模型生成、**还要扣一次免费额度**。额度该花在「这岗对我移民有没有用」上,不是「Ontario 是什么」。
- **排版不一致的根源**。2026-07-21 当天三个 bug 全出在这:公司面板渲岗位级匹配表、JD 首节标题作用域开大、黄条同屏堆两条——**根子都是同一套组件要伺候 24 种字段**,任何一处「按字段特判」都会漏。
- 参考站实测(E8 调研):Indeed 详情页零字段级弹框,信息按五行头部 + 分节铺开;Moving2Canada 二十步流程也是编号标题 + 段落,零弹框。

## 2. 拍板结果(2026-07-21,Frank)

**收成 3 个弹框**,对应用户真正会问的三件事(= 产品三问):

| 弹框 | 用户的问题 | 装什么 |
|---|---|---|
| **公司** | 这家靠谱吗 | 公司简介、行业、在榜职位、外劳记录(LMIA) |
| **职位** | 这活干什么、给多少 | JD 五节、薪资、雇佣形态、学历/证书 |
| **移民** | 这岗对我有没有用 | 对我意味着什么、PNP、EE、AIP、NOC/TEER、vs 中位、分步方案 |

**「移民」不并进「职位」**:它是付费核心(E5-00),并进去就埋没了。而 PNP/EE/AIP/NOC/TEER 现在各自一个弹框是散的——用户的问题从来不是「TEER 是什么」,而是「这岗对我移民有没有用」,这些字段本就该一起读。

## 3.5 统一卡片语言(全站,不止弹框)

Frank(2026-07-21):「手机端屏幕有限用卡片显示,浏览器端我觉得也最好用卡片,除非那种大表格可以保留。」

**但参考站实测(E8 调研)给出了一个更准的做法**:Indeed 详情页(手机版与桌面版)「有边框或阴影 + 圆角」的元素**命中 0 个**;Moving2Canada 正文区同样 0 个;Payscale 0 个;LinkedIn 仅少量且从不叠加。它们达成「分组清晰」靠的是**留白与字号阶梯**,不是描边盒子。

Frank 要卡片是为了「小屏下分组清晰」——**目的成立,手段修正**:

| 项 | 规矩 |
|---|---|
| 分节标题 | 20–26px 粗体,**在卡外**(不放进盒子里) |
| 分节间距 | 40–50px 纵向留白,**分隔靠空白不靠线** |
| 边框与圆角 | **二选一,绝不叠加**(要么 1px 直角线,要么 8–12px 圆角无边框) |
| 阴影 | 不用 |
| 一张卡 | **只装一个话题**,绝不塞第二件事 |
| 表格 | 仅当**需要横向对比**时用;手机上必须**全宽出血**(跳出正文左右内边距),否则挤成窄柱 |

**表格 vs 卡片的判据(拍板)**:**要横向对比就用表格,要逐条读就用卡片。**
职位列表 = 表格(用户在比不同岗的薪资地点,眼睛顺着列扫);「对我意味着什么」七行看着像表格,但**用户从不横向对比它们**,是一条条读判定 → 卡片(#158 手机版已改,桌面版待跟上)。

## 3.6 付费遮挡的统一形态

**地板规则:付费内容永不留空白,最低也要打码占位。**(Frank:「打码比直接不显示更能让用户有付费意愿」——空白=零信息、无失去感;打码=看得见摸不着,缺口才具体。)

三档,按「用户是否主动索取」决定:

| 档 | 场景 | 形态 |
|---|---|---|
| **档0 静默锁位** | 用户没主动要,只是路过看到 | 打码假值 + 灰字「Pro 解锁」,**无色无钮**。默认档 |
| **档1 行内提示** | 用户主动点了但额度用完 | 打码假内容 + 一行灰注 + **文字链**。**一屏最多一条**,其余同源降档0 |
| **档2 弹窗** | 用户点了升级钮 | 价格、对比表、卖点全在这 |

铁律三条:
1. **打码只能打在假数据上**。blur 是视觉效果不是访问控制,右键即可读 → 真值服务端剥离,前端渲假值。
2. **打码不需要真跑**。额度判定在调用之前,拦下就不生成(不预跑、不占朋友那台 qwen、不排队),糊掉的是假文本零成本。
3. **不许用假判定骗付费**。工资类可用假数字(只传达「有个数」的形状);**匹配度必须中性灰**——糊一个假「高匹配」= 拿假承诺换钱,付完发现是「低」就再不回来。真话留悬停(matchTotals 本就为真)。

黄色横幅**仅保留**给「整个功能不可用」(非某条内容被挡),因为那时确实需要一句人话解释。

## 3.7 三个弹框的内部骨架

沿用 `顾问弹框字段规范.md` 五段骨架,按新分组落到三个弹框(缺段跳过,不换序):

**公司弹框** ①页眉「公司」+ 公司名 → ③事实(AI 简介分节 / 行业 / 官网 / **外劳记录**) → ④在榜职位 → ⑤来源行
> **不渲「对我意味着什么」**(七维全是岗位级事实,挂公司标题下答非所问)。

**职位弹框** ①页眉「职位」+ 职位名 → ③事实(薪资、年薪折算、工时、雇佣期、学历、证书) → ④JD 五节整理版(首节标题跟容器走) → ⑤来源行(该岗 applyUrl)
> 薪资与年薪折算**合成一节**(原两个弹框说同一件事 = 重复排版)。

**移民弹框** ①页眉「移民」+ 岗位名 → ②**对我意味着什么**(判定在最上) → ③事实(PNP / EE / AIP / 职业分类五合一 / vs 中位) → ④AI 分步方案(三节) → ⑤来源行各归其源 → ⑥追问
> 付费核心在此。职业分类五级合成一节(#157 已证同名三级是噪音)。

## 3.8 AI 分步方案的产出规范(移民弹框 ④ 段)

三节:**【这个岗的移民信号】【分几步走(含时间线)】【怎么准备】**

硬约束(2026-07-21 四轮生产实测逐条踩出来的,每条都对应一次真实事故):

| 约束 | 起因 |
|---|---|
| 标题必须**描述性**不可是问句 | 「走哪条通道」逼出「此路径在移民法理上属于死胡同」= 违反红线 |
| **永不下资格判定**,不说能/不能移民、不用法律口吻 | 同上;移民资格是持牌顾问才能判的 |
| **只用喂进去的事实**,信号缺失不许编理由/名单/规则 | 编了「魁北克雇主担保名单」——本站根本没有 |
| **否定信号不许绕着编步骤** | pnp=false 的岗仍被排出「向省移民局提交 PNP 申请」 |
| 每步**必须以相对时间阶段开头**,只写动作视为无效 | 修英文标签时过度纠正,把时间线一起修没了 |
| 阶段名用**输出语言**书写 | 中文输出里冒出 "now – month 1" |
| 官方处理周期/费用**一律不编**,说「以官方公布为准」 | 用户会照着排计划 |
| 语言考试/ECA/证书落到**具体步骤**并说明哪些需申请前完成 | Frank:「什么时候考证,什么时候申请」 |
| 结尾给**本周该做的一件事** | 分析要能落到今天 |
