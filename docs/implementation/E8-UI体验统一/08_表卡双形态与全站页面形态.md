## 3. 通用卡片解剖规范(与 jobs 卡同基因,token 零新增)

```
┌────────────────────────────────┐
│ ①标题行  主实体名 14.5/600 蓝链   [右上操作位] │  ←可点(详情/官网/筛选);操作位=星标/对比钮
│ ②副行    次实体 12.5(别名灰字/Tag)          │  ←可省
│ ③键值区  label 11.5 灰 + value 13 深灰      │  ←两列 grid(数字短值)或一行一条(长值)
│ ④操作行  「看职位 →」12.5 蓝文字链           │  ←可省
└────────────────────────────────┘
```

- 容器:白卡 `#fff`+描边 `#e5e7eb`+r12,padding 10×12,卡间 gap 8(jobs 卡同规格);
- 数字语义色沿桌面列色:绿=薪资/技能股、棕=具名、灰=次要——**桌面表和手机卡同一数据同一颜色**;
- W 规矩:卡内禁「·」「/」杂糅,一键值一格;Tag/chips 用 primitives 现件;
- 可点规则=分元素点击(Frank 拍板):标题名/操作行各自可点,卡身不整卡响应;
- 排序:手机卡片流 v1 **不做排序控件**(桌面表头承担;手机默认序),记非目标;
- 转置表(compare/省对比):手机=**一实体一卡**——原「列头实体」变卡标题,原「维度行」变键值行。

## 5. 实现落点(DataTable 一处改,8 处标注)

- `DTCol` 加 `card?: 'title' | 'sub' | 'kv' | 'action' | 'hide'`(缺省:首列=title,其余=kv;label 复用作键值 label);
- `DataTable` 加 props:`cardBy?: 'row' | 'col'`(缺省 row;转置表传 col=一列一卡)+ `cards?: boolean`(occupations 传 false 走例外);
- 渲染:CSS 双渲染(`.dtCards`/`.dtTableWrap` + ≤640 media,jtCards 同手法零水合差异);卡片内 render() 原样复用(同一份单元格渲染函数,表/卡共用——**内容单一来源**);
- 8 处调用点:只加 card 角色标注,不改数据/render;
- i18n 零新键(label 全现成);验收=8 处手机断言(无横滚/角色映射对/分元素可点)+桌面表零回归。
