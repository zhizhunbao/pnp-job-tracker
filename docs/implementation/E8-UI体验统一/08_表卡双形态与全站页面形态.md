## 1. 抽象总账(2026-07-20 Frank 二拍改「按逻辑拆」:别都合并进一个组件——每个卡片内容排版都不一样)

| 层 | 是什么 | 覆盖 |
|---|---|---|
| **jobs 主表**(重器,独立) | 桌面满宽表+自带卡片流(#120 已双形态) | 1 处 |
| **DataTable**(公共表格) | **保持纯表格不动,不加卡片模式**(77 行的清爽不牺牲) | 桌面 8 处 |
| **卡片积木**(primitives 新增,纯样式原子) | `Card`(卡壳:白卡描边 r12+padding)/`CardKV`(键值两列 grid)/`CardAction`(操作行) —— 只管样式,零逻辑零配置 | 各域卡片共用 |
| **每域一个卡片组件**(与所在 View 同文件放) | EmployerLmiaCard / EmployerAipCard / RankCompanyCard / RankJobCard / StatsCatCard / CompareEmployerCard / StatsProvCard —— **各写各的内容与排版**,积木拼装 | 手机 7 处 |

原则:**组合复用样式,不合并逻辑**——通用的只有视觉原子;每张卡长什么样在自己组件里一眼可读,改一张不碰其余。
不收编:M 对比表(弹框语义表,已有窄屏堆叠)/模糊示例表×2(视觉道具)/邮件表(邮件模板)。

## 3. 通用卡片解剖规范(与 jobs 卡同基因,token 零新增)

```
┌────────────────────────────────┐
│ ①标题行  主实体名 14.5/600 蓝链   [右上操作位] │  ←可点(详情/官网/筛选);操作位=星标/对比钮
│ ②副行    次实体 12.5(别名灰字/Tag)          │  ←可省
│ ③键值区  label 11.5 灰 + value 13 深灰      │  ←两列 grid(数字短值)或一行一条(长值)
│ ④操作行  「看职位 →」12.5 蓝文字链           │  ←可省
└────────────────────────────────┘
```

- 容器:白卡 `#fff`+描边 `#e5e7eb`+r12,padding 10×12,卡间 gap 8(jobs 卡同规格);
- 数字语义色沿桌面列色:绿=薪资/技能股、棕=具名、灰=次要——**桌面表和手机卡同一数据同一颜色**;
- W 规矩:卡内禁「·」「/」杂糅,一键值一格;Tag/chips 用 primitives 现件;
- 可点规则=分元素点击(Frank 拍板):标题名/操作行各自可点,卡身不整卡响应;
- **hover 高亮规范(Frank 2026-07-20:「hover 要有高亮,用户才知道哪些能点」)——可点必有 hover 态,不可点必无**:
  ①文字链/标题链 hover=下划线+色加深(#2563eb→#1e40af);②可点单元格/行 hover=浅蓝底 #eff6ff(jobs 表 .jcell 先例,DataTable 行 hover 现为 #f9fafb → 统一升到 #eff6ff 仅当行可点,纯展示行保持无 hover);③钮/药丸 hover=描边转蓝;④纯文本/灰注/口径行=无 hover 无 cursor:pointer。触屏无 hover——手机端靠「蓝色=可点」的颜色语义承担(全站文字链二档定调 #108 已立);
- 排序:手机卡片流 v1 **不做排序控件**(桌面表头承担;手机默认序),记非目标;
- 转置表(compare/省对比):手机=**一实体一卡**——原「列头实体」变卡标题,原「维度行」变键值行。
