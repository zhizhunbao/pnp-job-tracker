> 缘起:2026-07-20 深夜 Frank 三连反馈:「详情页信息乱七八糟不知道重点」「点工作就是工作的内容,点公司就是公司的内容,不能混乱七八糟的」「哪些是开通 Pro 才能显示的,要重新设计」。

## 1. 三条铁律(整个详情页体系的宪法)

1. **一页一域**:职位页只讲这份工作;公司页只讲这家公司;统计页只讲地区/行业。跨域内容一律=一行链接跳过去,禁止整块嵌入(如:职位页里不铺公司简介,只给公司名链接)。
2. **一条信息只出现一次**:每个事实在页内只有一个「家」;第二次出现即 bug。
3. **Pro 位统一打码风格**(#130 先例):打码占位数(blur 写死假数)+「Pro 解锁」四字短注。禁长解释文案;真值不出服务端照旧。

弹框与详情页分工:**弹框=表内快看**(不离开上下文),**详情页=完整+可分享+SEO**;所有弹框标题栏恒有「打开完整页 ↗」(JD 弹框已有,公司弹框本批补)。

**去重清单**(现状 → 改后):
- 帖面=年薪时「薪资/年薪折算」两行同数 → 折算行+换算灰注仅时薪/日薪帖显示;
- meta 行「可省提名 NOC…TEER…」chip 与第 6 区重复 → chip 改锚点(点击滚动到通道节),不再自带展开内容;
- 城市在 meta 与事实块两处 → 只留 meta。

**Pro 清单(职位页,共 3 处,全部打码风格)**:vs 中位 ±%;匹配依据链明细(匹配药丸免费只显 高/中);拆解弹框超免费额度(402 升级卡照旧)。其余全免费(事实层免费红线)。
