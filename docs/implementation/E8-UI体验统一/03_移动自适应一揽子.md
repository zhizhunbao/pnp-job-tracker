# E8-03 · 移动自适应一揽子(所有组件手机可用)

> Epic **E8 UI 体验统一** · 负责人 Frank · 4 SP · 批次 B10
> 通用约定与索引见 [实现文档 README](../README.md)。2026-07-06 用户拍板:「所有的组件都要满足自适应,方便在手机上也能观看」。

---

## 1. 整体目标

全站(/jobs 主页 + 所有弹窗 + E8-02 弹窗化后的功能)在 **375px 视口**(iPhone 基准)可读可操作:不横向溢出页面、点击目标不小于 40px、弹窗窄屏自动全屏。桌面布局不回退。

## 2. 验收标准

- [ ] /jobs:顶栏/横幅/筛选区/搜索行 375px 不换行错位;表格在窄屏可横滚(冻结列保留)且页面本体不横向滚动。
- [ ] 所有弹窗:≤640px 视口自动全屏(含顾问弹窗——窄屏禁拖拽/八向拉伸,直接全屏);内边距收窄档统一进 Modal.tsx token。
- [ ] 筛选区窄屏折叠(常用筛选默认收起为一行「筛选」开关,展开抽屉式)。
- [ ] playwright 375/768/1280 三档截图走查清单过一遍(每页每弹窗),0 横向溢出。
- [ ] 触控:表头排序/列宽拖拽在触屏不误触(拖拽手柄 pointer events 已用 Pointer API,验证 touch)。

## 3. 实现步骤

- [ ] **3.1** Modal.tsx 加窄屏档:`≤640px → inset:0、radius 0、padding 收窄`(单一来源,所有弹窗自动获益)。
- [ ] **3.2** AdvisorModal 窄屏分支:直接全屏态,隐藏拖拽/缩放手柄。
- [ ] **3.3** /jobs 顶栏与筛选区 flex-wrap/折叠;搜索行与「更多筛选」合并为移动布局。
- [ ] **3.4** 表格容器 `overflow-x:auto` 复核(冻结列 sticky 在窄屏的表现;必要时窄屏减默认列)。
- [ ] **3.5** playwright 三档视口脚本(复用 E2E 工装;**记忆坑:preview 工具视口为 0,必须用 playwright**)跑全页面+全弹窗截图,人工过清单。
