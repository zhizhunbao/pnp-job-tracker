## 1. 整体目标

堵住上公网前的三个洞：① seed 是无鉴权 GET（`?reset=1` 可清库）；② advisor/jobtext 无鉴权可刷（LLM 成本暴露）；③ 密钥硬编码/弱口令。

## 2. 验收标准

- [ ] `GET /seed` 无 token → 401；带 `SEED_TOKEN` → 正常；build 容器日更 seed 不受影响。
- [ ] advisor：单 IP 超日限 → 429；全局超 `ADVISOR_DAILY_CAP` → 429；正常调用不受影响。
- [ ] `PAYLOAD_SECRET` / postgres 密码 / admin 密码全部强随机；`.env` 不在 git；`.env.example` 键齐全。
- [ ] README「上线安全清单」逐项过。

## 3. 实现步骤

- [ ] **3.1 seed token**：[seed/route.ts](../../../cms/src/app/seed/route.ts) 入口校验 `x-seed-token` header（或 `?token=`）=== `process.env.SEED_TOKEN`，缺失/不符 401；`etl/auto_update.py` build 角色的 curl 加 header（token 经 compose env 注入）。
- [ ] **3.2 advisor 粗限流**（Sprint 1 换按用户配额，本步是公测期防线）：route 内存 Map 按 IP 计数（日 N 次，env）+ 全局日计数；超限 429 带三语提示。jobtext 同样处理。
- [ ] **3.3 密钥轮换**：生成强 `PAYLOAD_SECRET`、postgres 强密码（compose 从 env 读）、admin 强密码；`.env.example` 补 README env 清单全部键。
- [ ] **3.4** 过一遍上线安全清单（README），逐项打勾留档。
