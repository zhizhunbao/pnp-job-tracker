## 1. 整体目标

堵住上公网前的三个洞：① seed 是无鉴权 GET（`?reset=1` 可清库）；② advisor/jobtext 无鉴权可刷（LLM 成本暴露）；③ 密钥硬编码/弱口令。

## 2. 验收标准

- [x] `GET /seed` 无 token/错 token → 401（实测）；SEED_TOKEN 未设=本地 dev 放行；build 容器 curl 带 `x-seed-token`（auto_update 已接）。
- [x] advisor/jobtext 限流实测：jobtext 限 2 → 第 3 次 429；advisor 限 0 → 立即 429（限流在缓存后、Ollama 调用前）。
- [~] 密钥轮换：`.env.example` 两份已重写齐全（cms/ + docker/）；**强随机实值在 VPS 部署时生成**（E2-01 3.2）。
- [ ] README「上线安全清单」逐项过（M1 收口时）。

## 3. 实现步骤

- [x] **3.1 seed token**：seed/route.ts 校验 `x-seed-token` header 或 `?token=`（SEED_TOKEN 已设则强制）；auto_update.py seed 请求带 header；prod compose 经 `${SEED_TOKEN:?}` 强制注入。
- [x] **3.2 限流**：新 `cms/src/lib/rateLimit.ts`（进程内日配额,多配额位原子判定）；advisor=IP 日限(默认40)+全局日上限(默认1000,LLM 成本兜底)、缓存命中不计数；jobtext=IP 日限(默认200)。超限 429（三语提示卡片随 E3-05 付费墙一起做,现为纯 429）。
- [~] **3.3 密钥轮换**：`.env.example`（cms/ 重写 + docker/ 新建）齐全；实值 VPS 部署时 `openssl rand -hex 24` 生成。
