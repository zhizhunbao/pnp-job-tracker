## 1. 原则(整件的宪法)

- **正文只留「导航性跳转」,知识性「官方来源」集中到一页。** 正文里逐条挂官方来源 ↗ = 杂乱、像薄聚合导流,掉信任观感;归拢到一页 = 正文清爽 + 官方来源没丢。
- **「官方可核」卖点靠归拢保住,不是删除。** 用户要核实/找官方入口 → 去 `/resources` 一页;这反而比散落更专业、更像"资源中心"。
- 顺序红线见 §5:**先建页,再清正文**。

## 3. 新页 `/resources`(官方资源导航)设计

- 路由 `/resources`,套 `PageShell`(1320 轨,与全站同宽)。
- **分类**(每类一卡;每条 = 名称 + 一句用途 + 官方外链按钮,按钮样式非蓝字链):
  1. **联邦移民** —— IRCC 主页、Express Entry、CRS 计算器、EE 历次抽选
  2. **各省 PNP** —— 10 省提名官网(职业清单/通道入口)
  3. **工资数据** —— ESDC Job Bank Wage、NOC 查询
  4. **LMIA / 雇主** —— open.canada.ca LMIA 季度表、AIP 指定雇主名单
  5. **权威与合规** —— canada.ca 移民主页、官方费用/处理时间
- **数据**:先硬编一份 **curated 常量清单**(静态整理,非 ETL);多数 URL 现成(各省 PNP 官网、canada.ca 本就在抓)。后续按需数据层化。
- **入口**:头部「资料库」下拉加一条「官方资源」+ 页脚;正文被撤的来源处不再逐条挂链。
- **SEO**:title「加拿大移民官方资源导航」;JSON-LD ItemList;进 sitemap。
