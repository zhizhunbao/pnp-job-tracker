## 1. 原则(整件的宪法)

- **正文只留「导航性跳转」,知识性「官方来源」集中到一页。** 正文里逐条挂官方来源 ↗ = 杂乱、像薄聚合导流,掉信任观感;归拢到一页 = 正文清爽 + 官方来源没丢。
- **「官方可核」卖点靠归拢保住,不是删除。** 用户要核实/找官方入口 → 去 `/resources` 一页;这反而比散落更专业、更像"资源中心"。
- 顺序红线见 §5:**先建页,再清正文**。

## 7. 公司区重排(讨论衍生,Frank 2026-07-23 效果图 `company_section_understand_first` 拍板)

**原则:先了解公司 → 再看移民价值**(Frank「用户应该先从了解这个公司开始」)。顺序:

1. **了解公司** = 公司名(「公司」标题已去,5030a05) + 下挂一行**中文「行业 · 做什么」**(真实行业字段,**非猜名字音译**——品牌名无官方中文,音译=瞎猜,沿用 [[value-thesis-real-data]] 城市名教训);有名的名字旁亮**「知名」章**(wikidata/wikiUrl 命中,现藏身份行 → 提到名字旁,信任秒建立)。
2. **公司介绍**(AI 五节/名录简介)—— **默认全展开**(Frank「默认都展开」;原提案"有名收起介绍"已否)。
3. **移民价值 · 会担保吗** —— 担保记录/档位下移到"了解"之后;无记录照实「无记录 ≠ 不担保」。

**定谳的几点**:
- 名字翻译**不做**(最初「公司名加中文」诉求 → 由"中文行业行"满足,既有中文又不瞎猜)。
- 有名/没名**不再区别折叠**(都展开),区别只在"知名"章 + 介绍详略由数据自然决定。
- **有名 ≠ 会担保**(大牌名企反而少担保,紧缺小公司才担保)——"知名"是信任信号,不替代担保信号。

**落地**:CompanyBody(弹框 + 详情页同源)重排;中文行业行取 company.industry / job.broad 大类中文;知名章 = company.wikiUrl 命中。触弹框 + 详情页 + 手机三处。
