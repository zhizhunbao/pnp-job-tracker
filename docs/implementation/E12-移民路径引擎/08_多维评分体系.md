# E12-08 多维评分体系(职位分×公司分×地区分)

> 缘起:2026-07-20 Frank「评分要拆开——公司评分、地区评分、职位评分,每个再细分维度,这样能找到最好的岗位和最好的公司」。
> 现状:职位有总分(08_score,按 TEER 一套权重)但**不可拆解**;地区有难度指数(E12-07 四因子,已上线)但**未进评分**;公司只有散信号(LMIA/AIP/知名/在库聚合)**无合成分**。
> 本件=设计文档;**未获批不动代码**。评分全部在**数据层算**(CLAUDE.md:分类/标签在数据层算,前端只读字段)。

## 1. 目标与非目标

- 目标:
  - ① **三层评分各自成体系**,每层带可拆解的维度明细(jsonb 存分项,difficulty jsonb 先例):
    - **职位分**(岗位级):移民通道 / 薪资质量 / 雇佣质量 / 雇主信号(继承公司分)四维;
    - **公司分**(公司级):担保记录 / 在库活跃度 / 薪资水平 / 规模知名度 四维;
    - **地区分**(省级):**直接复用 E12-07 难度指数四因子**,不另造(同一事实一套算法);
  - ② **不做跨层合成总分**(Frank 2026-07-20「岗位总分没啥意义」拍板):三层分各自独立、各答各的问题——职位分答「这岗本身值不值」、公司分答「这家雇主信号强不强」、地区档答「这个省卷不卷」;跨层拼盘掩盖驱动因素+权重任意=伪精确,禁;主表评分列语义收敛为**职位分**(自身维度合成,列位/排序兜底行为不变);
  - ③ **评分拆解弹框**:点评分格=维度×得分×依据 对比表(M 对比表同款观感),每维度带口径小注+出处;
  - ④ 「找最好的」出口:rankings 公司榜/职位榜接新分排序(现榜单已有壳);筛选区评分档筛照旧。
- 非目标:**不做「推荐/该选」话术**(分数=事实合成信号,选择权在用户,E6-02 红线);不做用户打分/评价(数据不做论坛);不新增抓取(全维度在库);不动 lib/match(「与我的匹配」是个人化信号,与客观评分分轨);不做城市/区级地区分(样本太薄,省级起步)。

## 2. 红线落地

- **粗筛信号,非资格认定**:每处评分沿用 pnpEligible 同款免责口径;拆解弹框尾行常驻;
- **维度可追溯**:每个分项在 jsonb 里带 `value/score/source` 三元(原始值+得分+出处),拆解弹框逐行可核——**没有黑箱总分**;
- **缺数不硬算**:某维度无数据=该维度不计入、权重重归一(留空不猜);公司无 LMIA 记录≠0 分担保维度,是「无记录」灰显(LMIA 语义红线:无记录≠不担保);
- **命名不带价值判断**:档位/榜单文案用「信号强/较强/一般」类中性词,禁「最好/优质/推荐」(找最好=用户排序自得,不是我们盖章);
- **裸「知名」标签退役**(Frank 2026-07-20「只显示知名我不清楚,能给各维度打分就好」):知名(wiki_url)降级为「规模知名度」维度依据之一,不再单独当 Tag 挂公司名旁;前端公司名旁改挂**公司分数字**,四维拆解在公司弹框——名录/榜单/对比表/卡片全站同步换装(存量知名 Tag 清理进本件)。
