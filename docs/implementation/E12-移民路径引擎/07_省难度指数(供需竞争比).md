# E12-07 省移民难度指数(供需竞争比 + 抽选水位)

> 缘起(2026-07-20 Frank):「一个省移民难度=看它的学签数量、工签数量、PNP 数量,数学方式算难度——这些数据我有吗?能算所有省吗?」
> 结论:**能算**。学签/工签/PNP 登陆数 = IRCC 开放数据全公开;PNP 年度配额 = 官方公告可抓;抽选水位/节奏/职业清单已在库(C6/B4)。
> 本件=设计文档先行(Frank:「先出设计文档,完了再考虑怎么做」),**未获批不动代码**。是 E12 三问平台「哪条路最快/难度多大」的第一块量化硬件。

## 1. 目标与非目标

- 目标:
  - ① **补抓四个 IRCC 数据域**(全官方、零爬虫对抗):学签持有者按省(月度)/ 工签持有者按省(TFWP+IMP,月度)/ PNP 登陆数按省(月度)/ PNP 年度配额(年度公告,一年 10 行);
  - ② **难度指数** = 供需基本面(竞争比)+ 抽选水位(果侧验证)+ 配额趋势,输出**三档分档(宽松/一般/紧)+ 逐因子依据链**,一省一行(QC 除外);
  - ③ 三个落点:stats 省页「移民难度」卡 → timeline 节奏块补配额行 → E12 方案卡/match 信号区(按此优先级分批)。
- 非目标:**不做成功概率**(伪精确红线);不做 QC(自有体系);不做 stream 级配额拆分(省不公开);不做权重自动学习;学校/雇主粒度不进 v1。

## 2. 红线落地

- **竞争基数 ≠ 申请人数**:学签+工签=潜在池粗口径(很多人不申请/不够格/会离开)。文案一律「竞争基数」「竞争比(粗口径)」,卡上灰字注口径;禁「排队人数」「申请人数」措辞。
- **分制不可比**:跨省横比只用纯人数量(竞争比/配额/节奏);**分数线只跟该通道自己的历史比**(水位=当前分在自身 24 个月分布的分位),沿 C6「非 CRS 标注」惯例。
- **历史统计不预测**:输出=分档+事实句(「近半年 3 天一抽、分数线降 12%、配额同比 -50%」),禁「你应该/你能成/成功率 X%」。措辞上线前 grep 复查(E12-01 同款)。
- **逐因子出处+数据月份**:每行带 IRCC 数据集 URL + asOf 月份;配额行带公告出处;沿「带出处带日期」铁律。
- **宁可留空**:某省某因子缺数 → 该因子不进合成、标「数据不足」;因子不足 2 个 → 该省不给总档,只列已有事实。

## 3. 数据 / schema

- **数据集存在性已实测**(2026-07-20,open.canada.ca CKAN API 命中):「Temporary Residents: Study Permit Holders – Monthly IRCC Updates」「Temporary Residents: Work Permit Holders」「Permanent Residents – Monthly IRCC Updates」三包全在;配额=IRCC 年度公告页(canada.ca httpx 直取,循 PNP/EE 抓取先例)。
- **raw(新增 etl/ 抓取,月更挂 ETL 轮同 ee 节奏)**:
  - `raw/ircc/study_permits.json` ← IRCC 开放数据「Study permit holders by province/territory of destination」(在学存量口径,**不是**新发签;口径写死进脚本注释);
  - `raw/ircc/work_permits.json` ← 同系列 work permit holders(TFWP/IMP 两列分存);
  - `raw/ircc/pnp_admissions.json` ← PR admissions by province & immigration category(取 PNP 行);
  - `raw/ircc/pnp_allocations.json` ← 年度配额:官方公告页抓取+人工核对(一年一批,带 source url + 核对日)。
- **clean(一个关注点一脚本)**:`etl/clean/04e_difficulty.py`(编号循清洗序)——读上面四份 + `raw/pnp/draws.json` + `raw/ee/draws.json`(已有),一次算清全部因子 → `processed/difficulty.json`。
- **mart**:`mart/difficulty.json`(一省一行,列对齐 DB)→ 09 合并 → seed 新表 `difficulty`(维度小表,白名单加表)。
- 因子公式 v1:
  - 竞争比 = (学签持有 + 工签持有) ÷ PNP 年配额;
  - 配额趋势 = 本年配额 ÷ 上年配额 − 1;
  - 抽选活跃 = 近 180 天抽选次数 + 平均间隔天数 + 邀请总量(C6 节奏同源逻辑,分组键=label 项目级惯例);
  - 分数线水位 = 各具名通道当前最低分在自身 24 个月分布的分位(通道级算,省级取具名通道中位)。

## 4. 分档规则(单一来源,常量表)

- 每因子先分三档,再合成总档;**阈值 v1 占位、首跑出全国分布后定**(阈值=代码常量单一来源,定档后回填本文档):
  - 竞争比:全国分布三分位切宽/中/紧(占位,待首跑);
  - 配额趋势:≥0 宽 / −30%~0 中 / <−30% 紧(硬事件:配额腰斩直接压总档到「紧」);
  - 抽选活跃:180 天 ≥8 次宽 / 3-7 中 / ≤2 紧(0 次=「休眠」单列);
  - 水位:分位 <40% 宽 / 40-70% 中 / >70% 紧。
- 合成优先级:配额硬事件 > 竞争比 > 抽选活跃 > 水位;规则纯函数进 `lib/`(消费端 stats/timeline/pathways 共用,不许 fork)。
- 输出 schema:`DifficultyRow { province, tier: 'easy'|'mid'|'tight'|null, factors: [{ key, value, tier, source, asOf }] }`。

## 5. 前端(批次落点,获批后分批做)

- **P1 stats 省页**:MetricCards 下「移民难度」卡(H 基础卡规格):档位 Tag(三语)+ 因子一行一条(W 规矩)+ 出处 ↗ + asOf 灰注;
- **P2 timeline**:节奏块补「年配额 X(同比 ±Y%)↗」一行;

- i18n 三语;人话名主文案+代码灰注(UI plain language 惯例)。
