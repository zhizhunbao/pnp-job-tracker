# E12-01 配方模型 + 旗舰①(直接就业→雇主担保→PNP)+ /pathways 方案卡

> 承接 `docs/移民路径方案引擎规划.md`(北极星,已定稿)P1。落点拍板(2026-07-18):方案卡=**新页 /pathways**
> (匿名=通用路径信息 SEO 落地页+建档 CTA;登录建档=个性化命中/缺口)。本件=配方 schema + 旗舰①配方 + 引擎核 + 方案卡最小 UI。

## 1. 目标与非目标

- 目标:① recipe schema(TS 结构化,政策数值**不硬编**——每事实带 source+lastReviewed,查不到留空「以官网为准」);
  ② 旗舰①配方 = 主线(offer→工签→省提名)+ **低门槛职业分支**(护工 HCW 试点 / 医护含 NNAS 认证红线 / trades 含 Red Seal);
  ③ 引擎纯函数(`lib/pathways.ts`,承接 match.ts「规则单一来源」):分型+档案 × 配方 × 现有维度 → 相关性 + 信号(带出处)+ 缺口→补法;
  ④ `/pathways` 页(SSR)渲染方案卡 + SiteHeader 入口 + sitemap 收录。
- 非目标:旗舰②留学路径/学校数据(E12-03)、政策源自动抓取核验+新鲜度告警(E12-04,本件 sources 为**人工核对**带日期)、
  顾问联动(E12-05)、匹配度打分(**故意不做**——评分会造「伪权威」,v1 只分「与你的处境相关/其他路径」+ 逐条 ✓⚠ 信号)。

## 2. 红线落地(§1 规划红线 → 实现约束)

- **摆信息不下结论**:所有 i18n 键措辞=「该通道要求…(出处↗)」「你的 NOC 在 X 省清单上」「你自报…」;
  **禁**「你应该/你就能/最适合你的是」。页顶+卡尾免责(信息非建议·非 RCIC·以官网为准)。
- **带出处带日期**:省清单/EE 命中信号用维度行自带 url+fetched(逐行出处);配方级 sources=官方页 URL+`lastReviewed`(人工核对日,E12-04 再进抓取核验)。**上线前所有 sources URL httpx 实测可达**。
- **宁可留空**:CLB 门槛/名额等数值 v1 不写(没抓)——写「语言等要求见官方页↗」。
- **受监管职业诚实**:医护分支步骤显式含「NNAS 评估+省护士协会注册」;绝不「有证就能工作」。
- **补法只指官方/可核实渠道**(IELTS/CELPIP/NNAS/Red Seal/IRCC ECA);缺 offer → **本站职位板深链**(筛担保信号岗)。不推中介、不背书第三方。

## 3. 数据 / schema

- **零 DB 变更、零 ETL 变更**。配方=`cms/src/lib/pathwayRecipes.ts` 结构化 TS(照 plan.ts 常量单一来源先例;编辑=改码,E12-04 再考虑数据化)。
- 引擎输入:`MatchProfile`(E11-04 含 currentStatus)+ `MatchDims`(pnpOccupations/eeCategories,行自带 url+fetched)。
- schema(示意):`PathwayRecipe { id, audience[分型], nocPrefixes?, steps[{key, sourceIdx?}], sources[{label,url}], lastReviewed }`;
  引擎输出 `PathwayEval { recipe, forYou, signals[{verdict,key,params,source?}], gaps[{key,fixKey,fixUrl}] }`。
- NOC 码红线:配方 nocPrefixes 涉具体码(44100/44101)逐条对照 `data/mart/noc_descriptions.json` 核验。

## 4. 引擎规则(lib/pathways.ts,纯函数)

1. **相关性**:currentStatus ∈ audience → 「与你的处境相关」;不在 → 「其他路径」(仍渲染,弱化);未设分型 → 全部平铺+建档提示。
2. **信号**(逐条 ✓⚠,全部可核验):省清单命中(nocCodes×targetProvinces×pnpOccupations named/excluded,逐行出处);
   EE 类别+上次抽选 vs 自报 CRS(照 match.ts 规则 3 口径);职业分支归属(nocCodes 前缀 ∈ 分支)。
3. **缺口→补法**(§7.1 表):未建档→建档;无目标省→选省;A 海外→「先拿能支持工签的 offer」+ 职位板深链;
   护士类 NOC→NNAS+省注册;trades→Red Seal;语言未报→官方考试页。
4. 规则只住这一处;UI/未来顾问联动都消费它的输出。

## 5. 前端(/pathways)

- SSR `pathways/page.tsx`(照 rankings 先例):getUser→profile→loadMatchDims→evalPathways→`<PathwaysView>`('use client',lang 循 LANG_KEY);generateMetadata 做 SEO(EN 标题+中文 desc)。
- 方案卡:名称+适用分型 chips / 有序步骤 / 你的信号(✓⚠ 逐条+出处↗)/ 缺口→补法(带官方链/职位板深链)/ 出处行(lastReviewed 日期)。
- 匿名:同配方通用态(无信号区)+「登录建档看你的命中」CTA;登录未建档:CTA → /account?sec=profile。
- 入口:SiteHeader 加「移民路径」(active 枚举扩 'pathways')+ 账户下拉深链;sitemap 收录 /pathways。
