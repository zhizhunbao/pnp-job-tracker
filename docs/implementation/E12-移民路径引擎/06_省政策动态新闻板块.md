## 1. 目标与非目标

- **P1 目标**:
  - **卡片流** `/news`:og 缩略图+原标题+AI 中文速读+日期+省筛选(全名 chips)+ ▶ 朗读钮;

- **非目标(P1)**:媒体新闻聚合(只官方源)、评论(P2/D6)、EN/KO 翻译层(原文就是英文;韩文翻译后续)、云 TTS(v1 平台特性 speechSynthesis 零成本,音质诉求出现再上云 TTS 缓存 mp3)。

## 2. 转载姿势(v2:转,姿势摆对)

1. **法律面(如实记档)**:联邦 Crown copyright 条款**非商业转载明确免许可**(注明出处+不冒充官方版本);商业场景灰区(条款称需询部门);各省 King's Printer 类似宽严不一。业内(移民媒体)整转+改写为常态,执法风险极低;真实风险=被当官方版本/旧闻当新政——靠声明+日期治理。
2. **姿势四件套**(每篇详情页):© 出处方(Government of Canada / 省 King's Printer)· 「非官方版本」声明 · 原文链接 ↗ · 官方发布日期。**沿用 E4-03 框架:转载+导流+异议即下架**。
3. **只官方源**;解析不出宁可缺省,一省失败不编不猜。
4. **AI 翻译/速读 grounding**:只喂抓取的官方正文,禁外推;标「AI 生成/翻译·以原文为准」;按 URL 幂等缓存(只对新增条目调 LLM)。
5. og:image hotlink(链接预览惯例)+ onerror 隐藏,不落盘缓存原图。

## 3. 解耦架构(Frank 点名:匹配 docker 拆分,互不影响)

**现有 compose 已是「一源一 service」**(SOURCE: jobbank/pnp/ee/enrich 各自独立 interval/healthcheck)——news 照此原生解耦:

```
docker-compose:  news 服务(<<: *etl, SOURCE: news, SCRAPE_INTERVAL: 43200=12h, 独立 healthcheck ping)
etl/sources/news/__init__.py   # 调度单元 META(method=httpx, seed=False 只刷 raw,build 轮消费)
etl/_scrape_base.py            # ★ 母脚本(#55 §2.5,2026-07-18 拍板):client/重试/去重/原子写/逐源隔离/防线——通用件全在这
etl/news/                          # 域目录(#55 新结构原生,不用迁)
  scrape_immigration_news.py       # 入口:母脚本驱动全部子源
  scrape_ircc_newsroom.py          # 子脚本:只填 SOURCE={region, list_url, kind, parse}(联邦 Atom 已实测 ✓)
  scrape_sk_sinp_news.py / scrape_ab_aaip_news.py / …   # 一省一子脚本,谁验证通过谁上
```
- **母/子分工**:母管怎么抓/去重/写盘/隔离/报数;子只填抓哪+怎么从该站 HTML 挑行(选择器优先、函数兜底)。news 是该模式**首个原生样板**,以后新源照此长。
- **命名规范(#55 §2.5,见名知意)**:`<动词>_<机构/源>_<数据内容>.py`——文件名单独出现(日志/报错栈)也自述抓哪家的什么数据。

- **三层互不影响**:① news 容器与其它源容器互不牵连(compose 级);② build_news 内逐省 try/except,单省源改版只丢该省(解析器级);③ 只刷 raw 累积文件,抓挂了旧数据还在(数据级,同 pnp 源惯例)。
- 这套「service+调度单元+域目录」三件套 = **#55 目录重组的原生样板**,以后新源照此长。

## 5. 数据模型 / 链路

- `raw/news/news.json`:**按 URL 累积去重**(同 postings 增量惯例);行={region(federal/SK/…), title, date, url, ogImage, **bodyEn(官方正文全文)**, **bodyZh(AI 段对段翻译)**, summaryZh(速读), fetchedAt}。
- **AI 翻译+速读**:build_news 内 python 直调 Anthropic(haiku,与顾问同模型;**新增条目才调**,翻译+速读一次调用产出,随行存 raw=幂等缓存);运维项:`ANTHROPIC_API_KEY` 进 docker/.env(ETL 侧首次用 LLM;成本≈每条新公告一次调用,公告频率低,忽略不计)。
- 09 直通 → `mart/news`(近 N=60 条,老的留 raw 不进站)→ 新 `News` collection(DDL 照 pnp_draws 形状+locked_rels 清单;bodyEn/bodyZh 为 text 长列)→ seed dims(白名单列全)。
- og:image + 正文抓法:逐条 GET 官方页抽 `og:image` + 正文容器(带频控;抓不到正文=只卡片不出详情页,不硬造)。
- **朗读**:前端 `speechSynthesis`(平台特性,零成本零后端);卡片读速读、详情页读全文翻译;暂停/停止;不支持的浏览器隐藏钮。
