## 1. 整体目标

`(frontend)/account` 三合一页（注册/登录/账户状态），三语，`/jobs` 顶栏挂登录态入口。

## 2. 验收标准

- [x] 注册 → 自动登录 → 账户态；顶栏 AccountLink 两态（登录/👤 用户名，客户端探测有 ~2s 异步延迟属预期）；登出恢复（本地全流程实测）。
- [x] /account 显示邮箱 + 免费版/⭐Pro·有效期至（SQL 模拟 webhook 拨 proUntil 实测）；购买按钮 E3-03 接。
- [x] 三语字典齐（zh 实测 + en 抽查；忘记密码=公测期占位文案，Resend 接入后补全）。

## 3. 实现步骤

- [ ] **3.1** `(frontend)/account/page.tsx`（client）：注册表单 `POST /api/users` → 成功即 `POST /api/users/login`；登录表单；登出按钮 `POST /api/users/logout`。
- [ ] **3.2** 登录态获取：服务器组件用 `getUser`（E3-01）传 props，或 client 调 `GET /api/users/me`——选服务器组件（少一次往返，且付费墙同路径）。
- [ ] **3.3** `/jobs` sticky 顶栏加 账户入口（未登录=「登录」，已登录=邮箱→/account）。
- [ ] **3.4** i18n 字典补 zh/en/ko 键。
