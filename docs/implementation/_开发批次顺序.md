> 配合 [工作项索引](README.md) 使用。把 24 个工作项按**依赖关系 + 办理卡点**排成批次。
> 建议 **一批 ≈ 一个 session**（上下文聚焦），一批收口再开下一批。
> **当前进度（2026-07-03）**：全部未开始，B0 待启动。

## 两类工作

- **核校**：代码**已存在** → 读码对照文档，修正文档漂移、列出缺口（本项目主要是 E1 合流前的回归核对）。
- **建**：代码不存在或不完整 → 按文档实现（本项目大多数工作项）。

> 每个工作项收口 = 文档勾选 + README 三张回归清单中相关项通过 + commit push。

## 执行原则

1. **B0 先行**：合流 + 数据回归基线，是后续一切的准确起点。
2. **安全与上公网同批**（B1）：seed token/端口收敛不做完，站点不挂域名。
3. **一批一 session**：上下文聚焦；批内工作项按文档顺序做。
4. **主线优先**：每批对应整体开发计划的单主线验收（M1→M2→M3→M4）；数据补强永远让路。
5. 手续类卡点**提前一批办理**（见上表）。
