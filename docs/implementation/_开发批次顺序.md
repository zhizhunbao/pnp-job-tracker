## 执行原则

1. **B0 先行**：合流 + 数据回归基线，是后续一切的准确起点。
2. **安全与上公网同批**（B1）：seed token/端口收敛不做完，站点不挂域名。
3. **一批一 session**：上下文聚焦；批内工作项按文档顺序做。
4. **主线优先**：每批对应整体开发计划的单主线验收（M1→M2→M3→M4）；数据补强永远让路。
5. 手续类卡点**提前一批办理**（见上表）。
