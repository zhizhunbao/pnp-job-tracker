# E6-06 职位类型筛选(全职 / 兼职 / 零工低门槛)

> 2026-07-17 Frank 提出:「职位筛选还要能知道哪些是 part-time、哪些是 full-time、哪些是可以打零工低门槛的工作」。
> 同批四问拍板「先①职位类型筛选」。**数据可行性当场实证**:Job Bank 详情页每岗都有结构化标注
> (`class="attribute-value"`:Permanent employment / Full time / Part time / Casual / Seasonal / Term or contract),
> 抽 80 份 2026-07-16 详情快照:Full 70 / Part 7 / Casual 1 / Seasonal 5 / Term 3——盘上 38k 份快照即是存量数据源,零新抓取。

## 1. 目标与非目标

- 目标:每岗两个新字段——**雇佣期**(permanent/term/casual/seasonal)与**工时**(full/part);
  筛选区加一个「职位类型」下拉:**全职 / 兼职 / 零工·临时**(零工=part ∪ casual ∪ seasonal,面向「低门槛先落脚」人群)。
- 非目标:不进评分/匹配规则(工时不是移民资格信号,别稀释 match 语义);不加表格新列(列已密,
  值放**弹窗事实块**;后续可按点击数据再议);ATS 岗不猜(JD 纯文本无结构化标注,宁缺留空——CLAUDE.md 红线)。

## 2. 数据 / 清洗(05b 原地加两字段,一个关注点一个脚本)

- **落点:`etl/clean/05b_parse_details.py`**——它已在解析详情页富集 processed postings,雇佣形态与它读同一段
  HTML(`attribute-value` 列表),term+hours 同源互依,按宪法第 3 条并入同脚本一次算清,**不新建脚本**。
- 解析:详情页 job posting attributes 区提取 → 归一化映射:
  `Permanent employment→permanent / Term or contract→term / Casual→casual / Seasonal→seasonal`;
  `Full time→full / Part time→part`(可并存,如 "Permanent + Full time");无标注/解析不到=空串。
- 新字段:`employment_term`、`employment_hours` 写回 processed/jobbank/postings.json(原地富集,幂等)。
- **存量回填(拍板点①)**:05b 增量语义=「没 `detail_fetched` 的才解析」,已解析的 38k 帖不会重跑 →
  加一次性参数 `--backfill-employment`:只重扫**已解析帖对应的 html 快照**提取两字段(不动 detail_fetched、
  不重写 .md),跑一遍即弃。快照已删的老帖=留空(宁缺)。
- ATS(`origin=ats`):两字段恒空;前端筛选选中任一类型时这些岗自然不命中(与「未分类」同一诚实口径)。

## 3. mart / DB

- `09_build_mart.py`:jobs 表透传 `employment_term` / `employment_hours` 两列(缺省空串)。
- Jobs collection 加两个 text 字段(有限值集但不用 select——与 broad/mid 等现例一致,校验在数据层);
  **生产 DDL 先行**(memory `prod-migration-workflow`):`ALTER TABLE jobs ADD COLUMN employment_term varchar, ADD COLUMN employment_hours varchar;`(纯新增列,无 rels 无索引需求——筛选在前端内存行上做)。
- **seed 列白名单同步两列**(老坑 5:改 collection 字段必同步 `seed/route.ts`)+ `lib/jobsList.ts` SELECT 带出。

## 4. 前端(一个 Sel + 三语)

- JobRow 加两字段;「更多筛选」区**移民资格行**后新增「职位类型」下拉(复用 `Sel`,贴内容宽度自动继承):
  选项 `full`(全职)/ `part`(兼职)/ `gig`(零工·临时=part ∪ casual ∪ seasonal;与 part 有交集,组合口径
  面向人群不面向字段,**拍板点②**)。谓词:`full`=hours==='full';`part`=hours==='part';
  `gig`=hours==='part' || term∈{casual,seasonal}。
- 弹窗:职位事实块加一行「职位类型:全职 · 长期」(两字段拼接,空则整行不显)。
- i18n:标签+3 选项+事实行,三语 ~6 键;saved-search 兼容(filters json 加 fEmp 键,jobsQuery 服务端谓词同步)。

## 5. 运维

- 无新容器无新调度:05b 在 build 轮既有位置,增量帖自动带出两字段;回填是一次性手跑。
- 数据观察:mart 后打印两字段非空率(预期 jobbank 详情已解析帖 ~100%,全库受 ATS/未抓详情帖稀释)。
