# E9-04 投递栏与注册闸(含求职意向表单)

> 2026-07-24 Frank 拍板(漏斗复盘:18 天 330 访客仅 1 注册 → 注册闸挪到全站意愿最强的「投递」瞬间):
> 详情底部常驻投递栏,点投递=收注册;注册后 3 组点选收求职意向,然后自动唤起邮件。
> 效果图已定稿:docs/assets/mockups/投递栏v2-20260724.png + 投递表单v4-20260724.png(v1 弹窗方案作废:字太多)。
> 范式:投递方式实时读原帖(复用 /api/jobtext 懒查询,禁批量预抓);记录复用 saved-jobs(E9-01);表单字段落 users 档案既有列。

## 1. 目标与非目标

- 目标:岗位详情底部常驻投递栏(大按钮「邮件投递」/「前往投递」+ 小按钮「复制要点」,无 Emoji,图标走 Icons.tsx);
  未登录点投递 → 注册框 → 求职意向表单(3 组点选,可跳过) → 自动继续投递,不让用户点第二次。
- 首版=「替他备好一切、他自己发」:mailto 预填收件人/主题/岗位信息。**不做代发**(邮箱授权/简历存储/发信信誉全后置)。
- 非目标:站内代发邮件、简历附件、投递状态跟踪(E9-01 状态标记已覆盖)、桌面端新布局(同一投递栏两端通用)。

## 2. 数据

- 无新表。投递记录写 saved-jobs(status=applied,已有值域);求职意向落 users 档案既有字段
  (职业方向→NOC 大类多选、目标地区→省多选、当前身份→新档案子字段,若需加列走 docs/sql 手写惯例,DB_PUSH 一律 N)。
- 投递邮箱不入库:点投递时从 jobtext 文本抽(`By email:`/mailto 正则,API 层与 jdformat 同位),抽不到=按钮回退「前往投递 ↗」外跳 applyUrl。

## 3. API

- 无新端点。jobtext 既有闸照走(402/429 语义不变);邮箱解析加在返回结构里(或前端对已拉文本正则,取实现最小者)。

## 4. 前端

- 详情面板底部 sticky 投递栏(效果图 v2):邮箱岗=「邮件投递」唤起 mailto;网站岗=「前往投递 ↗」;副按钮「复制要点」=岗位信息打包进剪贴板(标题/公司/地点/薪资/NOC/编号/原帖链接,用户贴给自己的 AI 也行)。
- 注册闸:未登录点投递 → AuthModal(reason 一句:注册后预填投递邮件并记录投递进度)→ 成功后弹求职意向表单(效果图 v4:职业方向/目标地区/当前身份全点选,选项读维度表 noc_categories+provinces,「暂不填写,直接投递」可跳过)→ 自动继续投递。
- 表单尾挂**可选**一行:「上传简历,自动完善档案(可跳过)」→ 复用 E11-07 解析自动填档案(学校/专业/经验从简历抽,不手填)。
  **2026-07-24 拍板(Frank 提议收姓名/生日/性别/学校/专业/经验,采纳反方案)**:匹配引擎不消费的字段不收;
  姓名/生日/性别=纯 PII 零匹配收益且加拿大求职语境敏感,永不进注册流;将来做代发/求职信时用到什么再收什么。
- 埋点:`apply` 事件(区分 email/web);隐私政策三语已含漏斗事件表述,无需再改。
- i18n 三语 `apply.*`。
