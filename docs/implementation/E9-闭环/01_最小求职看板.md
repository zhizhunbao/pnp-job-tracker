# E9-01 最小求职看板(收藏 + 状态标记)

> 2026-07-16 深夜 Frank 拍板「闭环吧,不然宣传用户也不知道怎么用」——冷启动前把行动环接上:
> 用户把求职进度存进站内 → 免费用户也有回访理由 → v3 六环「投递跟踪」的最小落地。
> 范式:全程照抄 SavedSearches(collection 访问控制/生产 DDL 手写/Payload REST 免写 API)。

## 1. 目标与非目标

- 目标:登录用户可对任意岗位收藏并标记状态(想投/已投/面试中/拿到 offer),在账户页「我的求职」管理。
- **收藏对免费用户开放**(区别于保存筛选=Pro):它是免费留存钩子,不是付费功能。
- 非目标(后续环):周报邮件(b)、分享卡(c)、成功案例回流(d)、手机卡片流收藏按钮(卡片寸土寸金,#8 拍板精神,先桌面操作列)。

## 2. 数据

- 新 collection `saved-jobs`:user(rel users, index)/ job(rel jobs, index)/ title·company(文本快照,岗位下架后看板仍可读)/ status(text:wish|applied|interview|offer)。
- status 用 text 不用 select:规避 Payload pg enum 类型,手写 DDL 更稳(值域由前端下拉限定)。
- 上限 `SAVED_JOBS_CAP=200`(plan.ts,env 可覆盖)防灌爆;创建 hook 强制 user=req.user。
- **不进 mart/seed**(用户数据表,与 users/saved_searches 同列)。

## 3. API

Payload REST 内置 CRUD(`/api/saved-jobs`),access=本人+admin,零自写路由(SavedSearches 先例)。

## 4. 前端

- /jobs 操作列加星标钮:未登录→开注册框(转化钩子);已登录→toggle(收藏=status wish / 再点=删)。
  已收藏集合登录后拉一次(`/api/saved-jobs?limit=200`)。
- 账户页 sidebar 新节「我的求职」:列表=快照标题@公司 + 状态下拉(PATCH)+ 删除 × + 「在职位板查看」链接(?q=标题)。
- i18n 三语 `sj.*`。

## 5. 生产 DDL(手写,先行;B7 教训:locked_documents_rels 关联列勿漏)

```sql
CREATE TABLE IF NOT EXISTS saved_jobs (
  id serial PRIMARY KEY,
  user_id integer REFERENCES users(id) ON DELETE SET NULL,
  job_id integer REFERENCES jobs(id) ON DELETE SET NULL,
  title varchar, company varchar, status varchar,
  updated_at timestamp(3) with time zone DEFAULT now() NOT NULL,
  created_at timestamp(3) with time zone DEFAULT now() NOT NULL
);
CREATE INDEX IF NOT EXISTS saved_jobs_user_idx ON saved_jobs(user_id);
CREATE INDEX IF NOT EXISTS saved_jobs_job_idx ON saved_jobs(job_id);
CREATE INDEX IF NOT EXISTS saved_jobs_updated_at_idx ON saved_jobs(updated_at);
CREATE INDEX IF NOT EXISTS saved_jobs_created_at_idx ON saved_jobs(created_at);
ALTER TABLE payload_locked_documents_rels ADD COLUMN IF NOT EXISTS saved_jobs_id integer;
CREATE INDEX IF NOT EXISTS payload_locked_documents_rels_saved_jobs_id_idx ON payload_locked_documents_rels(saved_jobs_id);
ALTER TABLE payload_locked_documents_rels ADD CONSTRAINT payload_locked_documents_rels_saved_jobs_fk
  FOREIGN KEY (saved_jobs_id) REFERENCES saved_jobs(id) ON DELETE CASCADE;
```

执行通道:backup 容器 psql(BACKUP_DATABASE_URI=Render 库)。
