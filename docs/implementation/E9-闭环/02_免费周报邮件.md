# E9-02b 免费用户周报邮件(收藏摘要)

> 2026-07-17 凌晨 Frank 拍板「先来文档」——闭环四素材的 (b):留存环缺口。收藏了岗的免费用户
> 现在没有任何东西拉他回站(邮件提醒=Pro 功能);发帖引来的注册用户 7 天不回访即流失。
> 范式:全程复用 E5-03 管线(alerts/run 双通道 + lastAlertAt 游标 + sendMail dry-run 语义),
> 本件=在同一路由里加**通道 C**,不建新任务不建新定时。

## 1. 目标与非目标

- 目标:**免费、有 ≥1 条收藏、距上次周报 ≥7 天**的用户,收到一封「你的求职看板本周动态」:
  收藏岗现状(几个还在招/几个已下架)+ 按收藏画像聚合的近 7 天新增岗计数 + 回站/建档/升级 CTA。
- **Pro 用户不发**(已有每日匹配日报,重复打扰反而掉价);无收藏的注册用户不发(没内容硬发=垃圾邮件)。
- 频率=滚动 7 天(人各有钟,不挤周一);随每小时 alerts 轮自然触发,游标控频。
- 非目标:邮件内容个性化到岗位行级明细(v1 只给计数+链接,点回站里看)、分享卡(c)、成功案例(d)、
  给匿名订阅邮箱(无账号体系外的 newsletter 不做)。

## 2. 数据

- **users 加两列**(生产 DDL 手写先行,B7 惯例):
  ```sql
  ALTER TABLE users ADD COLUMN IF NOT EXISTS last_weekly_at timestamp(3) with time zone;
  ALTER TABLE users ADD COLUMN IF NOT EXISTS weekly_opt_out boolean DEFAULT false;
  ```
  Payload Users collection 同步加 `lastWeeklyAt`(date, admin 只读)/ `weeklyOptOut`(checkbox, 本人可改)。
- 内容素材全部现表:saved_jobs(该用户收藏+快照)join jobs(status 在招/closed)+ jobs 近 7 天
  datePosted 按「收藏画像」(收藏岗的省/大类去重集合)聚合计数——**不新增任何画像存储**,现算。

## 3. API

- `api/alerts/run` 加**通道 C(weekly)**:查询 = 免费(无 proUntil 或已过期)× `weekly_opt_out=false` ×
  `last_weekly_at` 空或 <now-7d × 存在 saved_jobs;逐人算内容 → `sendMail` → 成功才回写 `lastWeeklyAt`
  (dry-run 不发不回写,与 A/B 通道同语义);响应体加 `weeklyEmails` 计数。
- **退订(CASL 合规,必须免登录一键可达)**:新 `GET /api/alerts/unsub?u=<userId>&t=<token>`,
  `t = HMAC-SHA256(PAYLOAD_SECRET, 'unsub:'+userId)` 截断 16 hex——无新密钥、无新表;校验过则置
  `weeklyOptOut=true`,返回极简「已退订」页(纯文本级)。邮件 footer 两条路:一键退订链接 + /account 开关。
- 防超时:单轮通道 C 发送上限 200 封(游标天然分片,下一轮续),与 seed→alerts 180s 超时预算兼容。

## 4. 前端

- 账户页「我的求职」节尾加一行开关:「每周邮件摘要」checkbox(读写 users.weeklyOptOut,语义取反显示);
  三语键 `sj.weekly` / `sj.weeklyOff`。
- 邮件模板(zh+en 双语一封,与 E3-07 口径一致):
  标题「你收藏的 N 个岗:X 在招 · Y 已下架 · 你的方向新增 Z 岗」;正文三块=收藏现状表(快照标题+在招/下架)/
  新增计数行(「SK·科技 近 7 天 +37」,链接=/jobs 预套筛选)/ CTA(免费=「建档案看匹配」,已建档=「升级看全量」);
  footer=退订链 + 免责一句。**红线:已下架岗不写「已关闭无望」类结论,只陈述状态**(粗筛信号非资格认定同族口径)。

## 5. 生产 DDL 与运维

- DDL 见 §2(backup 容器 psql 通道执行;users 表无 locked_rels 新关联,俩标量列无 B7 坑)。
- 触发链零改动:auto_update seed 成功 → alerts/run(现有)→ 通道 A/B/C 顺序跑;
  healthcheck/日志照旧,日志行加 `weekly:N`。
- 发量预估:冷启动期注册用户 <100,周报 ≤14 封/天,Resend 免费档(100/天)富余;超量再谈。
