# E9-03 地区冷启动横幅(当前地区优先·软引导)

> 2026-07-17 凌晨 Frank 提出「应该跟进用户 IP 优先显示当前地区的职位」。
> 产品拿捏:**「优先显示」做成软引导(横幅一键套筛选),不动默认排序**——
> ① 展示约定拍板过默认排序=发布时间(CLAUDE.md),这是日更职位板的根;
> ② PNP 用户常看的是**别的省**(人在 ON 看 SK/AB 通道是常态),按地区硬重排会伤移民视角差异点;
> ③ E9-02 推荐横幅已有「引导→一键套筛选」的完整壳,本件=给它补一个**冷启动信号源**(首访无浏览画像时)。

## 1. 目标与非目标

- 目标:**首次访问、还没有浏览画像**的用户,顶部横幅显示「你所在地区:ON · 看这些岗(N) / 建档案精确匹配」,
  一键套省筛选;浏览画像成型(ev≥5)后**画像优先、地区退位**(真实行为 > 位置猜测)。
- 非目标:改默认排序/默认筛选(全国最新在前不动)、服务端按 IP 分内容(SEO 同 URL 同内容,别给爬虫看地域版)、
  地区弹窗打断(横幅够了)。

## 2. 定位信号:浏览器时区,不用真 IP(拍板点①)

- **推荐:`Intl.DateTimeFormat().resolvedOptions().timeZone` → 省码映射**,纯客户端:
  - 零新依赖(Render 不注入 geo 头;真 IP 方案=引 GeoIP 库/mmdb 月更 或 调外部 API=新外部依赖+每访问一跳);
  - **零上传零 PII**——与推荐横幅「本地画像不上传」同一隐私口径,隐私政策不用改(IP 定位则要加条目);
  - 海外用户(Asia/* 等)映射不到省 → 横幅自然不出,不用特判「人在国内看加拿大岗」。
- 映射表(TZ → 省):Toronto→ON / Vancouver→BC / Edmonton→AB / Regina·Swift_Current→SK / Winnipeg→MB /
  St_Johns→NL / Moncton→NB / **Halifax→NS(近似:大西洋时区横跨 NS/NB/PE,取人口主省——拍板点②:
  可接受近似,或按「宁可留空」不映射大西洋时区)**;白名单外(含 NT/YT/NU 与非加时区)→ 不显示。
- 精度够用的理由:横幅是**推荐不是事实断言**,套错省一键可改/当日可关;时区差错率远低于它带来的首访破冰价值。

## 3. 数据 / API

零。纯前端,无新存储(沿用 `jobsPrefHide` 当日关闭键;不新增 localStorage 键)。

## 4. 前端(一个 useEffect 分支 + 前缀文案)

- `JobsTable` 横幅 useEffect:现逻辑 `if (pf.ev < 5) return` 改为——ev<5 时查时区映射,命中省则
  `setRec({ prov, broad: '', sal: '', src: 'geo' })`;画像路径照旧(`src: 'pref'`)。
- 渲染:`rec.src === 'geo'` 时前缀用新键 `rec.geoPrefix`(「你所在的地区:」/ "Jobs near you: " / "내 지역: "),
  CTA 两枚与关闭逻辑全复用;`n===0`(该省今日无岗,几乎不可能)沿用不显示。
- 三语新键 1 个(`rec.geoPrefix`);#51 的 loadedAll 数字口径自动继承。

## 5. 运维

无。不落任何服务端状态;umami 若要看点击率,后续给 CTA 加事件(非本件)。
