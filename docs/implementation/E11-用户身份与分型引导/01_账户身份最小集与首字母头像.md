# E11-01 账户身份最小集 + 首字母头像

> 承接 `docs/用户信息与社交登录规划.md`(定稿)§3.1 身份层 + §5 头像。E11 第一件、无外部依赖,是分型/引导/E12 的地基。
> 拍板:头像 v1 = **首字母块 + OAuth 头像 URL,不做上传**;displayName 可选(默认取邮箱前缀)。

## 1. 目标与非目标

- 目标:Users 加 **displayName / avatar / loginProvider / locale** 四个身份字段(均本人可改);
  首字母默认头像组件;账户页显示头像+昵称,可改昵称。
- 非目标:上传头像(YAGNI)、Google/微信登录(E11-03/06)、账户下拉改版(E11-02)、分型 currentStatus(E11-04)。

## 2. 数据 / schema

Users collection 新增(均**无字段级锁**=本人可改;role/proUntil/stripe* 仍锁死):
| 字段 | 类型 | 说明 |
|---|---|---|
| `displayName` | text | 昵称;空则前端回退邮箱前缀 |
| `avatar` | text | 头像 URL(v1 只存 OAuth 带回的 URL;无则前端画首字母块)。**存 URL 不做上传** |
| `loginProvider` | text | 'email'（默认）/后续 'google'/'wechat';注册来路 |
| `locale` | text | 'zh'/'en'/'ko';界面/邮件语言(现仅 localStorage,落库备后续邮件用) |

- schema 推送:`DB_PUSH=1 npm run dev` 一次(加 4 个可空列,additive 安全;直连生产库,先确认无其它 collection 漂移)。Payload 同步 payload-types.ts。

## 3. 后端 / API

- 无新端点。字段本人可改走 Payload 自带 `PATCH /api/users/:id`(update 已限 selfOrAdmin;profile 同款)。
- loginProvider 默认 'email':注册端点不特殊处理,字段默认值给 'email'。

## 4. 前端

- **首字母头像组件 `Avatar`**(新,`(frontend)/Avatar.tsx`):有 `src` 则 `<img>`;无则首字母色块(名/邮箱首字母 + 由字符串稳定 hash 出的颜色)。尺寸参数化。
- **账户页**(account/page.tsx overview):显示 `Avatar` + displayName(空回退邮箱前缀)+ 邮箱;昵称可就地编辑(input + 保存,PATCH /api/users/:id { displayName })。
- Me 类型加 displayName/avatar/locale。

## 5. 运维

- 无。一次性 schema 推送记入本文档落地记录。
