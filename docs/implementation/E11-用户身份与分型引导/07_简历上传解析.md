# E11-07 · 简历上传解析(立项,待 Frank 拍板)

> 状态:**立项文档,未开工**。付费复审(20260719)队列首位:建档转化杠杆——档案匹配是付费核(E5-00),
> 但建档要用户手填 NOC/CLB/CRS,摩擦大;简历一传自动预填,转化链路缩短一半。
> v3 六环定位:这是「简历环」的地基,后续解析建议/投递/模拟面试都踩在「简历已在库」上。

## 1 · 目标
用户在建档 wizard(或账户页)上传简历(PDF/DOCX)→ 自动抽取:工作经历(职位名+年限)、教育、语言线索
→ 职位名映射 NOC 候选(在库 NOC 词表)→ **预填档案表单给用户确认**(不静默入库——沿清洗铁律:宁可留空不瞎猜,
简历抽取是「建议」不是「事实」)。确认后走既有 E5-00 档案保存,匹配立即生效。

## 2 · 链路(拟)
```
wizard 第 1 步新增「上传简历,10 秒自动填」(可跳过,手填路径保留)
→ POST /api/resume:multipart 收文件(≤5MB,PDF/DOCX)
→ 文本抽取:pdf 用 pdf-parse / docx 用 mammoth(纯 node,Render 上可跑,零外部服务)
→ 结构化:LLM 单次调用(引擎=决策题①)抽 {经历[{title,years}], 教育, 语言成绩}
→ 职位名 → NOC 候选:在库 noc 词表模糊匹配,给前 3 候选让用户挑(不自动定)
→ 前端:wizard 各步预填,用户过目确认 → 保存(既有 /api/profile 不动)
```

## 3 · 关键决策(★=要 Frank 拍板)
- **★① 解析引擎(隐私红线优先)**:简历=高敏 PII(姓名/电话/履历全在)。
  - A. **Anthropic API(Haiku)**:~$0.001/份,商业条款不训练,隐私口径可写进政策页;**推荐**。
  - B. 朋友 qwen(ngrok):零成本,但简历发第三方个人服务器——隐私上**不可接受**,只配翻译公开新闻那类。
  - C. 纯规则词表:零成本零外发,但中英简历版式千姿百态,抽取质量撑不起「10 秒自动填」的卖点。
  - 推荐 A;若 Frank 不想引入 API 账单,可 C 起步 A 升级(接口同形)。
- **★② 原件存不存**:
  - 立项版=**只解析不存原件**(解析完即弃,库里只落用户确认后的档案字段+解析原文快照 hash 供排错)——隐私最小化,零存储成本。
  - 二期(投递环需要简历本体)再上 Supabase Storage+删除自助;到时补隐私政策条款。
- **★③ 免费口径**:上传解析=免费(它是转化杠杆,不是付费点;付费仍在匹配列/无限匹配)。防滥用:登录后可用+每账号 5 次/天。
- ④ 语言成绩:简历极少写 CLB——抽到雅思/思培分就换算预填,抽不到留空让用户填(不猜)。
- ⑤ 失败回退:任何一步失败 → toast「解析没成,手动填也很快」+ 正常手填流程,不阻断。
