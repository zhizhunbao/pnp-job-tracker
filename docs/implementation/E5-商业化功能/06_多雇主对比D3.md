> 本件=设计文档(molit 惯例开工前出);**未获批不动代码**。

## 1. 目标与非目标

- 目标:
  - ① `/employers/compare` 对比页:2–4 家雇主并排,维度×雇主对比表(M 对比表同款观感,DataTable 转置列先例);
  - ② 入口三处:公司弹框「加入对比」钮 / /employers 名录行勾选 / 对比页内搜索补位(pg_trgm 公司搜已有);
  - ③ 免费态=⑤价值时刻先例(三行价值点+模糊示例表+升级钮);Pro 全开;
  - ④ 建档 Pro 用户附「与我的匹配」行(按我的 NOC/目标省逐雇主出 match 信号)。
- 非目标:不做雇主评分/推荐排序(「你该选 A」禁——摆事实高亮差异,选择权在用户);不做评论/口碑(产品哲学:数据不做论坛);不新增抓取(全部维度已在库);不做分享链接 v1。

## 2. 红线落地

- **LMIA 语义红线(E6-02)**:LMIA=雇过外国人的历史事实 ≠ 担保承诺;AIP 指定 ≠ 有配额——口径行写死,对比表表头 tip 同款措辞。
- **摆事实不下结论**:逐维度对比+差异高亮(最优值绿标),**禁**「推荐/更好/该选」;卡尾免责(非资格认定·以官网为准)。
- **出处随行**:LMIA→ESDC、AIP→指定名单、难度→IRCC(E12-07 卡同源)、AI 简介→K 调查出处;每维度行带 ↗。
- **缺数留空**:某雇主某维度无数据=「—」,不猜;AI 简介缺=触发 K 懒调查(现有管线,谁被对比谁被查)。

## 3. 数据(零新抓取,全在库)

| 维度 | 来源 | 备注 |
|---|---|---|
| LMIA 获批职位(总/技能股/最近季度) | companies.lmia_* | 技能股列=担保信号主维度(#B4-02 口径) |
| AIP 指定 | designated_employers 匹配 | 大西洋四省 |
| 在库在招岗数 / 平均分 / 具名通道命中岗数 | jobs 聚合(rankings sponsor-likely 同口径) | SQL 现算或复用榜单行 |
| 行业 / 知名 / 中韩别名 | companies.industry / wiki_url / alias_* | 雇主 D 富化 |
| AI 调查简介+官网 | companies.ai_*(K 缓存) | 缺=懒触发,同一缓存 |
| 所在省移民难度档 | stats.difficulty(E12-07) | 多省经营的雇主取主要省(在库岗数最多省) |
| 帖面中位薪资 | jobs 聚合 | 对照用 |
| 与我的匹配(建档 Pro) | lib/match 现有纯函数 | 逐雇主取其在库岗的 match 分布(高/中命中数) |

- **服务端 gate**:对比数据 API 仅 Pro 返回真值(compare 先例:免费不回传真数据,示例值硬编码前端)。

## 4. 交互 / 形态

- 选择态:公司弹框/名录行「加入对比」→ 顶部浮条「已选 N 家 · 去对比」(localStorage 存 slug,≤4 家,超提示);
- 对比页:表头=雇主名(别名/知名 Tag 同名录);行=上表维度;差异高亮=数值型维度最优值绿字(仅事实高亮,非推荐);
- 空态/免费态:⑤先例复刻(价值点:LMIA 记录并排看/担保信号一眼比/按我的档案逐家看匹配 + 模糊示例表 + UpgradeCta);
- 入口权重:公司弹框钮=主入口(选 offer 场景在弹框里);/employers 勾选=browse 场景。
