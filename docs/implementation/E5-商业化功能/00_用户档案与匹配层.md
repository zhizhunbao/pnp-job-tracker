# E5-00 · 用户档案与匹配层（付费墙头牌）

> Epic **E5 商业化功能** · 负责人 Frank · 7 SP · Sprint 1-2 之交 · 批次 B5
> 依据 [产品重设计提案](../../产品重设计提案.md) §3（2026-07-03 用户采纳）：**档案匹配取代 AI 顾问成为付费核**。
> 通用约定与索引见 [实现文档 README](../README.md)。

---

## 1. 整体目标

把 PRD v0.1 丢掉的「对**我**移民的价值」找回来：用户建档案（NOC/CLB/CRS/目标省/PGWP 余量）→ 逐岗计算「与我的匹配 + 差在哪 + 依据链」。**零新增抓取**——匹配 = 档案 × 现有维度（pnp_occupations / ee_categories / ee_draws / wages）的运行时 join。

## 2. 验收标准

- [x] 登录用户可建/改档案（字段见 3.1），未建档案看到引导。
- [x] Pro 用户列表出「与我的匹配」列（高/中/低/不适用 + 点开弹依据）；弹框加「对我意味着什么」块，**每条结论带依据链**（指回具体维度记录，如「EE STEM 上次抽 491 · 2026-06-xx，你自报 480 → 差 11 分」）。
- [x] **措辞红线**：只说「符合/不符合 XX 公开清单条件」「高于/低于上次抽选分数线」，**永不说「你能/不能移民」**；块底带免责短句 + 来源行（复用 E4-04 机制）。
- [x] AI 顾问档案感知：Pro 用户的 advisor system 追加档案事实（grounded 契约不变，问到档案没有的信息直说未提供）。
- [x] 免费层激活钩子：建档案免费，每日前 N 岗可见匹配（N 在 plan.ts）。
- [x] 固定 fixture 档案 × 抽样岗位的匹配结果快照测试（规则改动可回归）。

## 3. 实现步骤

- [x] **3.1** Users.ts 加 `profile` group 字段：`nocCodes[]`（经验/学历 NOC）、`clb`(number)、`crs`(number,自报,可空)、`targetProvinces[]`、`pgwpMonthsLeft`(number)、`profileUpdatedAt`。重启 dev 推 schema。
- [x] **3.2** `(frontend)/account` 加档案表单（NOC 选择器复用 noc_categories 维度做搜索下拉；三语）。
- [x] **3.3** `cms/src/lib/match.ts` 纯函数：`match(profile, job, dims) → { level, reasons: [{verdict, evidence, sourceRef}] }`。规则 v1（各规则独立、可单测）：
  - NOC 对口：job.noc ∈ profile.nocCodes（或同小类）；
  - 省通道：job 省的 inclusion/exclusion 表 × job.noc × profile.targetProvinces；
  - EE 距离：job.eeCategory 的最近抽选 CRS − profile.crs（ee_draws 维度）；
  - TEER 可达：teer ≤ 3 或命中低 TEER 通道；
  - 工资信用：salaryAnnual vs wageMedAnnual（offer 可信度提示）。
- [x] **3.4** 列表接线：page.tsx 服务端对 Pro 用户逐行调 match（当前页行数级计算,性能无虞）→ 新列 + 排序支持；免费用户按 plan.ts 限额。
- [x] **3.5** 弹框「对我意味着什么」块（FieldFactsSection 同级）+ advisor system 档案事实注入。
- [x] **3.6** fixture 快照测试 + 三语文案 + 红线措辞自查。

## 7. 实施记录(2026-07-04)

- **规则引擎 = `cms/src/lib/match.ts` 一处**(纯函数、前后端同构):五规则各自出 reason(i18n 键+参数+sourceRef 指回维度记录),加权分→高(≥60)/中(≥30)/低/不适用;v1 不用 clb/pgwpMonthsLeft 评分(档案存着、advisor 事实可见)。快照测试 tests/int/match.int.spec.ts 10 项,含**措辞键白名单审计**(新增 reason 键必过一遍红线)。
- **列表**:服务端逐行算 level(Pro 全量/免费前 N);SQL 排序补 score 兜底与前端默认序对齐——否则同日大量并列时「前 N 岗」≠ 用户看到的前 N 行(实测踩到,已修)。
- **弹框「对我意味着什么」**:客户端用同一 match() 重算依据链(dims 已在 props),✓/⚠/✗ 逐条 + 来源链接(label↗,title=抓取日期)+ 免责短句;放 FieldFactsSection 上方。
- **advisor 档案感知**:Pro 且建档 → system/facts 注入自报档案 + 本岗匹配结论(reasonEn 同源英文行);**个性化初判缓存按人隔离**(缓存键加 :p<uid>,防串人)。
- **偏离文档一处**:NOC 选择器数据源用 `noc-descriptions`(有码+官方名,397 行)而非文档写的 noc_categories(该维度只有分类名没有码);表单在 /account,支持码/职业名搜索+手输 5 位码。
- 本地 15 项端到端全过(档案 CRUD/回读、字段锁回归、SSR 匹配列、免费前 10 chips + 11 起锁、Pro 全解锁);生产 users 表 6 个 profile 列已先行补齐(psql,用户授权)。
