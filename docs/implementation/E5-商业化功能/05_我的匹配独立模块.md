# E5-05 · 「我的匹配」独立模块(付费入口化)

> Epic **E5 商业化功能** · 负责人 Frank · 2 SP · 起因:2026-07-06 深夜用户拍板
> 「匹配不该是表格里的一列(未建档时 50 行重复『建档案→』纯噪音)——应该是**单独模块**,
> 用户付费之后点进去,自动显示与自己匹配的 job。」

## 1. 目标

把付费核(E5-00 档案匹配)从「混在 29 列里的一列」升级为**有自己入口的目的地**:
顶栏「我的匹配」→ 点进去就是「匹配我的岗」的列表,匹配等级排最前,零配置。
主表默认列里**删除 match 列**——建档引导只留页头横幅一处(07-06「别到处都是」拍板的延续)。

## 2. 方案(Ponytail:零新查询,视图复用)

- **不是新页面**:/jobs 加 `?view=match` 视图模式——同一张 JobsTable、同一个 fetchJobRows
  (match 服务器早已按人算好),前端过滤 `match != null && match != 'na'` + 排序 匹配等级↓ → 发布时间↓。
  URL 可分享、可回退;窄屏行为照旧(E8-03)。
- **顶栏入口「我的匹配」**,按登录态分流:
  - 未登录 → 登录弹窗;已登录未建档 → 档案页/建档引导;
  - 已建档 → 进入 view=match(能看多少由下面 D1 拍板决定)。
- **主表变化**:match 从默认列与列选择器中移除(视图模式里它固定第一列);
  「建档案 →」逐行 CTA 随之消失。
- **模块内空态**:建档但零命中 → 「暂无命中,你的档案是 NOC xxxx × CLB x × 目标省 X;
  放宽目标省/更新 CRS 试试」+ 改档案链接(诚实,不硬塞低相关岗)。
