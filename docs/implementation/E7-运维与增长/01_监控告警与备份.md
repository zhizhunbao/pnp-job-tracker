## 1. 整体目标

三层监控（ETL 心跳 / 站点存活 / LLM 用量）+ 每日库备份。全部用免费托管服务，代码入侵仅「成功后 ping 一行」。

## 2. 验收标准

- [ ] 每个 ETL 源（jobbank/pnp/ee/build）有 healthchecks.io check：正常轮次 ping 到；**人为停掉一个源 → 超时邮件告警收到**。
- [ ] UptimeRobot 监控 `/jobs`：人为停 caddy → 宕机告警收到。
- [ ] Anthropic Console 用量告警已设（E2-03 项复核）。
- [ ] 每日 pg_dump 落盘保留 14 天；**恢复演练**：从昨日备份恢复到临时库 → 行数一致。

## 3. 实现步骤

- [ ] **3.1** healthchecks.io 建 4 个 check（周期=各源 SCRAPE_INTERVAL + 宽限）→ URL 进 env `HEALTHCHECK_PING_<SOURCE>`。
- [x] **3.2** `etl/auto_update.py`：每轮全部 steps 成功后 `httpx.get(ping_url, timeout=10)`（env 缺省则跳过——本地开发不 ping）。
