## 0. 设计要点

1. **清洗下沉**:检测=`etl/clean/visa_flag.py` 纯函数(一个关注点),由 09 在「JD 正文下沉」同一循环调用(不另起脚本重扫 44k 文件——重复解析反模式)。
2. **精确优先宁可漏**(误伤=帮雇主赶走本可投的人):两档红旗 `no_sponsorship` / `pr_required`;三道护栏=「legally eligible to work」样板句不算(任何工签都满足)/ or·and 逃逸句(「or hold a valid work permit」)不算 / 福利条款(「benefits… PR only」)不算。全量 44,235 个 JD 实测:94+12 命中,人工过全部样本,护栏迭代两轮后零可疑残留。
3. **quote=命中原句**(≤180 字)一路带到弹框——citation 惯例,用户可核验,不是黑箱标签。
4. **「—」口径写死**:未检出≠雇主提供担保,只是 JD 没写(弹框 note + 筛选 tip 双处)。
5. **seed 过渡防线**:ETL 盒未 git pull 期间 mart 无此字段——upsert 用 COALESCE 保留旧值不清空。
