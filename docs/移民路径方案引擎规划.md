# 移民路径方案引擎 —— 北极星规划

> 2026-07-17 Frank 提出:要能**识别并生成具体方案**——例:朋友想让孩子一毕业就能拿工签→PR,
> 「去海洋省(Halifax)读电焊,读完本地就业,直接申 PR」= AIP 大西洋移民计划。每省每地区政策不同,
> 要先从官网找到**最新政策**,再按人(分型)给出「选学校/选专业/选工作」的方案 + 对比数据。
> 拍板(本次两问):① **深度先行**——先把 1–2 条旗舰路径做透;② 学校数据**先范围化**——只做旗舰路径相关。
> **本文档=北极星规划(架构+数据模型+旗舰范围);具体政策数值不写死,由抓取层带出处带日期填(反幻觉红线)。**

## 0. 这是什么(一句话)

给用户的处境(E11 分型 A–E + 目标),输出**具体、可执行、带出处的移民路径方案**:
「读什么专业 → 在哪个省的哪类学校 → 毕业后做什么工作 → 走哪个通道 → 怎么申 PR」,附匹配度、缺口、和对比数据。
—— 这是项目「三问决策平台 / 跨四级门槛代劳」价值主张的**正餐**:不知道有哪条路 → 不会查官网 → 看不懂政策 → 要一个能照着走的方案。

## 1. 红线(最重要,决定这产品是护城河还是雷区)

1. **信息 ≠ 建议(非 RCIC)**:我们摆**官方、带出处、带日期的路径信息 + 用户匹配度信号**,让用户自己决定;
   **不**做「你照这走就能拿 PR」的个性化法律建议(持牌顾问地界)。沿用项目既有免责口径。
   - **定位自证(2026-07-17 Frank)**:IRCC/RCIC 查的本就是公开官方资料 → 我们也能全查到、保持最新;
     没牌照不能给建议,但「基于数据匹配 + 提供信息 + 用户自决」不需要牌照——差别全在**措辞**:
   - **可以(摆信息)**:「AIP 要求指定雇主 offer(出处↗,抓取 2026-07)」「你的 NOC 在 SK 紧缺清单上」
     「这条通道要求 CLB X,你自报 Y」「命中/缺口如下,信息仅供参考,请以官网为准」。
   - **不可以(下结论/代理)**:「你应该走 X」「你照这做就能拿 PR」「最适合你的方案是…」
     「我帮你把申请填了/递了」「按你情况我判断你能移民」。凡是「你应该/你就能/我判断」→ 越线。
   - **收费口径**:卖的是**数据/匹配工具**(Pro=信息访问),不是「移民咨询服务」;避免任何「付费买建议」的表述。
   - **上线前**:路径引擎比职位板更贴近「建议」边界,**发布前建议做一次面向加拿大移民法的措辞审阅**(我非律师,这是稳妥的去风险步,不是法律意见)。
2. **带出处带日期**:每条政策事实挂官方链接 + 抓取日期(承接 E4-04 字段级 citation);政策会变,过期要能看出来。
3. **不幻觉、宁可留空**:查不到的数值留空并说明,绝不凭记忆/推测编(尤其语言分、名额、成功率——**永不编成功率/保证**)。
4. **对比数据只从权威源**:薪资=ESDC(已有);学费/学制=学校官网;不采无源的「网传」数字。

## 2. 用户视角(输出长什么样)

一条**方案卡**(示例结构,数值占位=待抓):
```
【路径:大西洋移民计划(AIP)· trades 通道】            匹配度 ●●●○ / 缺口:暂无本地 offer
你的情况:B 在加留学(或 A 海外直申需先入学)
步骤:
  1) 在 NS/NB/PE/NL 读认可院校的电焊等 trades 文凭(≥X 年 · 出处↗ · 抓取 2026-07)
  2) 毕业后拿到「指定雇主」全职 offer(本站有 3766 家指定雇主可筛 · 见 AIP 数据)
  3) 省背书 + 语言达 CLB X(出处↗)+ 定居计划 → 递 PR
对比:该职业本地薪资中位 $X/yr(ESDC)· 学制 X 年 · 学费 $X(校方)
出处:IRCC AIP 页 ↗ · 该省 AIP 页 ↗（均带抓取日期）
```

## 3. 四层架构

| 层 | 内容 | 现状 |
|---|---|---|
| **① 政策配方层** | 每条路径=结构化「配方」(见 §4);带出处+日期 | 已有 PNP 清单/EE 类别/AIP 雇主 → 扩成完整配方 |
| **② 学校/专业层** | 旗舰路径相关的学校+专业(DLI、PGWP 可申、trades) | **新数据域,范围化建** |
| **③ 对比层** | 薪资(ESDC 已有)/学费·学制(校方)/职业紧缺 | 部分有 |
| **④ 规则/匹配引擎** | 分型+目标 → 命中路径、排序、步骤、匹配度、缺口 | 建在①②③上,复用 match.ts 思路 |

## 4. 政策配方数据模型(pathway recipe)

一条路径抽象成结构化配方(字段示意;**数值留空待抓,不硬编**):
```
pathway {
  id, name(AIP-trades / 省国际毕业生 / EE-FSW …),
  region: [省码…],                          # 适用地区
  audience: [分型 A–E],                       # 谁能走
  study_requirement { needed?, level, min_years, eligible_program_types, recognized_institutions_ref },
  job_requirement { offer_needed?, employer_type(指定/一般), noc/teer 约束, seasonal? },
  language { clb_min_by_teer },               # 待抓,带出处
  other { 定居计划?, 省背书?, 名额/开放状态 },
  steps: [有序步骤],
  sources: [{ url, publisher, fetched_date }], # E4-04 citation
  last_reviewed: date,
}
```
—— 引擎读配方 + 用户档案 → 判「命中/缺口」,渲染成 §2 的方案卡。

## 5. 两条旗舰路径(深度先行;按「不留学 vs 要留学」的根本分叉,最现成的放 P1)

### 旗舰 ① 直接就业 → 雇主担保 → PNP(不留学·省时间 —— 2026-07-17 Frank 补的核心场景)
- 即「雇主 offer → 省提名」——**项目的原始核心**(CLAUDE.md 立项即此),故**数据最现成、零新数据**:
  现有全国职位板 + `pnpEligible` 信号 + LMIA 外劳雇佣记录 + AIP 指定雇主,**全部已在库**。
- 受众:A(海外,要拿能支持工签的 offer 再登陆)· D(在加找担保岗)· C(在职换更好担保岗)。
- 步骤:筛能担保的岗(直发 / 有 LMIA 记录 / 指定雇主 / 命中省清单)→ 拿 offer → 工签 → 攒经验 → 省提名 → PR。
- 缺口高亮:A 海外 →「先拿到能支持工签的 offer」;语言/职业清单达标与否。
- **职业专属低门槛通道(2026-07-17 Frank:年纪大/英语弱/不想读书/想做护工·国内护士)**:同是「不留学」,职业门槛差极大——引擎要按人推**对**的职业通道,而非硬塞需高 CRS/英语的 EE:
  - **护工 / PSW**:联邦 Home Care Worker 试点等,语言/学历门槛相对低 → **最适合年纪大 / 英语弱 / 不读书**人群的现实落脚点。
  - **医护 / 护士**:多省有医护紧缺专属通道(项目**已有** `sk-health` / `ns-critical` 数据)。
  - ⚠️ **诚实红线**:护士是加拿大**受监管职业**——国内证**不能**直接执业,须先过 **NNAS 评估 + 省护士协会注册**(credential recognition)。
    引擎要如实摆出「先做资格认证」这一步 + 出处,**绝不简化成「有证就能直接工作」**(假希望=踩真实数据红线)。
  - 分型联动:英语弱/年纪大/不读书 → 引擎**优先推低门槛职业通道**(护工/部分医护/trades),EE 退位。
- **为什么 P1 首选**:零新数据域,纯把已有职位板 + PNP/LMIA/AIP/医护紧缺信号**组织成一条「不留学」路径 + 配方话术**,最快出价值。

### 旗舰 ② 留学 → PGWP → 省提名(要读书的路径)
- 覆盖最多在加留学生(B);各省国际毕业生/CEC 通道(配方按省抓)。
- **AIP 大西洋 trades(电焊例子)= 这条的高价值具体实例**(大西洋 + trades 文凭 + 指定雇主)。
- 依赖:PGWP 可申 DLI 子集(**范围化新建**)+ 各省毕业生通道配方 + 大西洋 trades 院校。

> 两条覆盖「不留学 / 要留学」根本分叉;都接 **E11 分型**,按处境只推相符路径 + 高亮缺口(A 海外→先拿 offer / B 留学→先毕业)。

## 6. 学校/专业数据(范围化)

- **旗舰①(不留学)不需要学校数据** —— 又一个它做 P1 首选的理由。
- 学校数据**只**为旗舰②:大西洋四省 trades 院校/专业 + 各省 PGWP 可申 DLI 子集。
- 字段:院校名 · 省 · 专业/项目 · 学制 · PGWP 可申? · 官网 · 学费(有则收)· 抓取日期。
- **不**一上来建全加拿大所有 DLI(YAGNI;范围化,旗舰②要哪些收哪些)。

## 7. 规则引擎(④)

- 输入:用户分型(A–E)+ 目标(职业/省/时间线)+ 现有移民档案(NOC/CLB/CRS…)。
- 逻辑:遍历配方 → `audience/region/study/job/language` 逐项比对档案 → 命中(全满足)/ 潜在(缺 1–2 步,给缺口)/ 不适用。
- 输出:命中路径按匹配度排序 + 每条的步骤/缺口/对比/出处。规则集中一处(承接 match.ts「规则单一来源」)。

### 7.1 缺口 → 补法(2026-07-17 Frank:不止说「差什么」,还给「怎么补·在哪·要什么证·靠谱渠道」)

每个未满足项映射到**官方/可核实**的补齐路径(带出处),把「缺口」从报问题升级成给方案:

| 缺什么 | 怎么补(只指官方/可核实渠道) |
|---|---|
| 语言不够 | 考 IELTS / CELPIP(法语 TEF/TCF);给目标分 + 官方报名页 |
| 学历认证 | IRCC 认可的 ECA 机构(WES/ICAS/IQAS…) |
| **护士/医生执业**(受监管) | 护士:**NNAS + 省护士协会注册**;医生:MCC + 省医学院注册 |
| trades 技工证 | 省学徒/技工认证、**Red Seal** |
| 缺担保 offer | 用**本站职位板**筛能担保的岗(直发 / LMIA 记录 / 指定雇主 / 命中省清单) |
| 缺工作经验 | 该通道认哪种经验、要几年(配方里带,带出处) |

- **按职业给对应补法**:护工≠护士≠医生≠电工,不同 NOC 的执业/认证要求不同——引擎按岗位的职业属性只给**那一套**。
- **红线**:只指向**官方/可核实机构**;**不推荐私人中介、不为任何第三方背书**;**绝不涉及「买证/代考」等造假**(违法+毁信任)。
  仍是「信息 + 出处,用户自己去办」,不代办不下结论。

## 10. 风险 / 诚实

- **政策churn**:程序时开时关(名额/关停/新通道),官网散。需持续抓+标日期+人工复核关键项;引擎对「开放状态/名额」要能显示「未知/以官网为准」。
- **配方正确性**:移民规则细、例外多;旗舰路径也要逐条对官方核,标注「粗筛信号非资格认定」(沿用现有口径)。
- **学校数据维护**:新数据域,范围化控住量;别失控扩成全 DLI 目录。
- **不越线成 RCIC**:全程「信息+出处+匹配信号」,不出个性化法律结论。
