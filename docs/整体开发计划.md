## 一、项目概述

PNP Job Tracker 是每日更新的**全加拿大全职业职位板**，差异点是**移民价值视角**：每个岗位叠加 PNP 具名通道 / 联邦 EE 类别 / AIP 指定雇主 / vs 工资中位 等信号并按移民价值评分，配 AI 顾问弹框（事实 / 判断 / 对话三层，grounded 不编造）。目标用户是走「雇主 offer → 省提名」路线的在加求职移民者。

**当前状态（起点，详见 STATUS.md）**：ETL → mart → seed → /jobs 全链路本地跑通；库内 ~4800 岗 / 3600 公司；Docker 自动日更（jobbank 2h / pnp 周更 / ee 月更）；前端三语（中英韩）、全字段筛选、固定列表格、AI 顾问弹框。**尚未有**：云端部署、用户账号、支付、合规页面；AI 顾问依赖家里 Ollama（线上不可达）；代码在 `feat/lists-autoupdate-and-table-ux` 分支未合 main。

**核心设计原则（不变）**

- Ponytail 有纪律的极简主义：能不写的代码不写，收费功能同样过决策阶梯（Payload 自带 auth 就不引第三方；Stripe Checkout 托管页就不自建账单 UI）。
- 清洗下沉数据层：seed 只入库、前端只显示，商业化不破坏这条铁律。
- 数据真实性高于覆盖率：宁可留空不瞎猜；AI 顾问只基于喂入事实说话。
- **移民信息合规红线**：本站是信息聚合工具，不是持牌移民顾问（RCIC）意见——免责声明是上线前置条件，不是收尾项。

## 二、目标与成功标准

**总体目标**：2026-08-28 前把本地已跑通的系统变成**可公开访问、每日自动更新、可完成订阅付款**的正式产品，并在 9 月入学前收敛为无人值守运行。

**成功标准（验收口径）**

- 公开域名可访问，`/jobs` 首屏 < 2s，移动端可用。
- 云端 ETL 每日自动更新，连续 7 天无人工干预数据保持新鲜。
- AI 顾问在线上可用（云 LLM），成本有测算、有限流。
- 用户可注册 / 登录，可完成一笔**真实的订阅付款**并解锁付费功能，可自助退订。
- 免费 / 付费分层生效：免费层可浏览核心列表（引流），付费层解锁 AI 顾问等高价值功能。
- 合规四件套上线：免责声明（非 RCIC）、隐私政策、服务条款、数据来源标注；雇主个人联系方式不公开展示。
- 监控告警在位：ETL 失败、seed 失败、站点宕机能通知到人。
- 北极星（首月参考值，不作硬性验收）：注册 ≥ 50，付费 ≥ 5。

## 三、系统架构

```
etl/ (Python)                          cms/ (Payload + Next.js)
raw(抓取) → clean/ → mart ──seed──▶ Postgres ──▶ /jobs 公开页(免费层)
  │ 云端常开主机跑容器(现有 docker/)      │           ├─▶ 付费层: AI 顾问·提醒·完整信号
  │ jobbank 2h · pnp 周 · ee 月          │           ├─▶ 榜单页(SEO 引流, F8)
  └─ 政策清单: PNP/AIP/EE 自动更新       │           └─▶ 账号(Payload auth) + Stripe 订阅
                                    云 LLM API(替家里 Ollama, 按量+限流)
```

- **架构不变**：两段式、数据展示严格分离照旧；上线 = 把现有 docker compose 搬到常开主机 + 托管 Postgres，不重构。
- **新增的三块**：① 账号与订阅（Payload 自带 users/auth + Stripe Checkout + webhook 写订阅状态）；② 付费墙（前端按订阅状态 gate 功能，API 侧同步校验）；③ 云 LLM（AI 顾问后端从 `OLLAMA_URL` 换成云 API，接口不变）。

## 六、团队与节奏

单人项目（**Frank**，全栈 + Claude Code 辅助）。

**故事点（SP）口径**：沿用 MOLIT 计划——1 SP ≈ 半个工作日（约 4 小时），**8 SP / 周**为基准，余量留给运营、学习支付/部署新领域的摸索成本。全程 4 个 Sprint ≈ 64 SP。

单人项目的排期纪律：**每个 Sprint 的验收标志只有一条主线**（上线 / 收款 / 开闸 / 收敛），数据补强类工作永远给主线让路——移民信号数据已经够收费，缺的是「能收钱的通路」。

## 八、风险与阻塞（重点）

> ⚠️ 与 MOLIT 不同，本项目无外部交付方，**风险集中在合规、平台依赖与单人带宽**。数据链路是已验证的存量能力，风险最低。

| 风险 / 阻塞 | 影响 | 等级 | 归属 | 应对 |
|---|---|---|---|---|
| 移民信息合规：被视为提供移民建议（RCIC 持牌监管） | 法律风险，动摇收费正当性 | 🔴 高 | 外部 | 免责声明前置到 Sprint 1 就挂站；话术全站统一「信息工具，非移民建议」；AI 顾问输出内嵌提示 |
| 抓取数据 republish：Job Bank ToS / JD 版权 / 雇主 PII | 下架风险、投诉 | 🔴 高 | 外部 | 只展示职位事实 + 官方投递链接；JD 摘录不全文转载；个人联系方式不出前台；收到异议即下架机制 |
| Job Bank 改版 / 反爬升级 | 数据断供 = 产品停摆 | 🟡 中 | 外部 | 抓取失败告警（Sprint 3）；raw 快照留存可回放；解析层独立便于快速适配 |
| 云 LLM 成本失控（AI 顾问按量计费） | 毛利倒挂 | 🟡 中 | 内部 | 只对付费用户开放 + 每日调用限额；选低价模型；上线前按定价核算单用户成本上限 |
| 单人带宽：8 周 + 9 月入学硬截止 | 延期即撞开学 | 🟡 中 | 内部 | 每 Sprint 单主线验收；Sprint 3 整体可降级;缓冲周不排新功能 |
| 支付/部署是新领域，摸索成本难估 | Sprint 0/1 超时 | 🟡 中 | 内部 | 全部用托管方案（Stripe Checkout、托管 Postgres），不自建；卡壳 2 天即换备选 |
| 付费意愿未验证（免费替代品多） | 收入不达预期 | 🟡 中 | 外部 | M1 先免费公测收反馈；定价低门槛试水；免费层保留引流价值 |
| 家用机 → 云主机资源差异（内存/磁盘） | ETL 容器跑不动 | 🟢 低 | 内部 | Sprint 0 上云即压测一轮完整 ETL；不行升配（成本可控） |
