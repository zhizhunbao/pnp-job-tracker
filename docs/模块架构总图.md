## 2. 新功能归属判定(三问)

1. **动的是哪层数据?** 新抓取/清洗 → A;只读已有字段 → 产品面。
2. **服务用户旅程哪一环?** 找岗 → B;身份决策 → C;账号/我的东西 → D;掏钱 → E。
3. **是采集入口还是资产运营?**(D3 vs E13 的分界,简历上传的教训)采集喂 profile → D3;围绕资产本身做功能 → 独立模块。

**归位先例**:「我的收藏」→ D4(不是新模块);「简历上传建档」→ D3/E11-07(不是 E13);「LinkedIn 登录」→ 不做(API 拿不到简历,受众不符);「省份全名」→ D3 通俗易懂原则(#58)。
