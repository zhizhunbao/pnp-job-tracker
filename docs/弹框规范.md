## 2. 内容骨架(自上而下的固定顺序)

所有「内容类」弹框按同一顺序组织,缺哪块跳过哪块,**不换序**:

1. **eyebrow + 标题**(ModalTitle;eyebrow 颜色=场景色:顾问靛蓝 #6366f1、升级琥珀 #92400e)
2. **个人相关块**——「对我意味着什么」/建档引导(有档案:匹配结论+依据链;无档案:建档 CTA 条)
3. **事实块**——字段级真实数据;官方数据在前、本站推断在后
4. **AI/动态内容**——生成内容直接开始,**不带任何免责/AI 声明**(2026-07-06 用户拍板:合规统一在页面 footer,弹框内零合规文案——advisor.disclaimer、match.disclaimer、footAI、chatTitle、chatHint 全部移除);流式时尾部 ▋
5. **来源行**——底部一行极简出处:`来源: <完整 applyUrl>` 可点击、新窗口,**不带**发布方/抓取时间/「官方原帖」标签(2026-07-06 用户拍板,取代 07-05「链接 chip 行」与「全移除」两版;E4-04 field_sources 维度与 /sources 解释页保留)
6. **升级卡/错误态**——402 → UpgradeCard;错误 → 红底提示条(样式同表单错误:#fef2f2 底 #fecaca 边)

表单类弹框(登录/注册、升级)例外:品牌头居中(登录保留 🍁+站名,用户拍板)或标题行,之后是表单/按钮,错误条样式同上。

## 4. 新增弹框守则

- 居中弹框一律 `<Modal>`,不许自绘 overlay;需要自由定位才允许自绘,且必须消费 Modal.tsx 的 token
- 内容按 §2 顺序放;来源=底部一行「来源: 完整 URL」(§2-5),合规文案零出现(footer 统一)
- 措辞红线不变:事实+出处,永不说「你能/不能移民」
