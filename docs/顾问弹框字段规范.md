> 本文档 = 弹框的「显示宪法」:哲学 → 统一骨架 → 九个字段家族逐族规范 → 排版格式 → 待拍板决策。**拍板前不动代码。**

## 1. 哲学(四条,前两条是既有原则的成文化)

1. **三层零混淆(信任产品的骨架)**:事实(绝不经 LLM,来自 DB/维度表)→ AI 判断(只吃上面的事实,禁编数)→ 追问(同一 grounding)。**层与层必须视觉强分界**——用户一眼分清「机器算的」和「模型说的」。这次「$72K-$78K 编数」事故的教训:AI 文与事实块视觉揉在一起时,编的数看起来像真的。
2. **值 → 口径 → 来源 三件套**:每个事实块自带「这个数怎么来的」(口径注)+「出处」(SourceLine:官方原帖优先/数据集兜底 + 抓取日)。**缺数据明说,且分两种诚实**:「该 NOC×省真无数据」≠「Pro 功能」(已落地 fact.noMedian/medianPro 先例,推广到所有 Pro 列)。
3. **字段家族统一骨架**:29 个 ColKey 归 **9 个家族**,同家族同结构同顺序。用户学会读一个弹框 = 会读全部。新字段入列时先归家族,禁止再发明第十种排版。
4. **field 级与 job 级分区**:弹框正文只放**本字段**的内容;job 级的东西(建档 CTA、投递页/官方原帖按钮)固定在恒定位置(页眉/页脚),不与字段内容抢位、不随字段变化。

**措辞红线(全局,既有)**:粗筛信号非资格认定;永不说「你能/不能移民」;省分数带分制标注「非 CRS」;LMIA=「雇过外国人的历史事实」非「能担保」。

## 2. 统一骨架(每个弹框五段,固定顺序)

```
① 页眉   [AI 顾问 · 字段名]  大标题=本列单元格值   + job级动作链(投递页↗/官方原帖↗) + 全屏/关闭
② 对我   「对我意味着什么」:建档=等级+逐条依据链(match() 同源);未建档=单行CTA;限额外=升级卡
③ 事实   段头「事实」+ FactsBox(K-V行/清单/高亮命中)+ 口径注 + SourceLine(合并进框底)
④ AI 判断 段头「AI 判断」(免责小字合并进段头行)+ 流式正文(【标题】分段;SIMPLE族一句话)
⑤ 追问   chat 输入框 +「仅基于上方事实回答」提示
```

- ①的大标题**只取本列自己的值**(已修:元素类 cell 不回退别列文本);取不到就不显示大标题,页眉 tag 已有字段名。
- ②③④⑤간 用分隔线+段头强分界;③④之间是「机器/模型」边界,分界最强(建议:④整段浅底色或左边框)。
- 空段规则:某段无内容整段消失(不留孤儿段头);③永不为空(最少一行本列值)。

## 4. 排版与格式规范

- **段头**:12px/700/灰 `#6b7280`,lucide 图标 + 文字(事实=IconPaperclip、AI=IconCompass、对我=IconTarget、追问=对话图标),下边距 6。
- **FactRow**:键 12.5px 灰,值 13.5px 深灰 `#374151`;数值 `tabular-nums`;null 值整行隐藏(不显示「—」占位行,少即是多)。
- **口径注**:11.5px 浅灰,框内底部;**必须与内容匹配**(有中位才说 ESDC 口径——本次事故点)。
- **chip 语义色全局恒定**(弹框=列表同色):琥珀=具名/命中,绿=通过/可行,蓝=EE/联邦,紫=QC,灰=不适用,红=排除/下架。
- **清单**:>200px `overflowY: auto`;命中行琥珀底+「← 本岗」+自动滚进视野(既有,不变)。
- **免责**:全弹框**只出现一次**,合并进④段头行(现状两处:AI 区顶部 + 底部 footAI —— 保留哪处见 D2)。
- **AI 正文**:14px/1.7;【标题】加粗分段;SIMPLE 族一句话不分段;流式光标 ▋。
- **动作链**:job 级(投递页/官方原帖)进①页眉右侧或恒定页脚条(见 D3),不再插在④与⑤之间。
