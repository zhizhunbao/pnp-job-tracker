环节行为
首次打开某岗 JD先立即显示原文 + 顶部细条「AI 整理中…」→ 服务端拿 description 调 /api/chat(五节模板 prompt)→ 校验过 → 存 jobs.jd_formatted(+时间戳/模型标) → 前端切整理版
后续任何人再开直接读库,零等待零调用——被看过的岗永久受益,没人看的岗永不浪费算力
顺手补字段同一次调用捎带抽 {工时, 雇佣期},岗位缺这俩字段的就地补上(622 缺失岗的兜底也惰性化,不单跑批)
降级ngrok 掉线/超 10s/校验不过 → 静默保持原文,不报错不重试轰炸;同岗并发去重(in-flight 锁)
红线①prompt=只搬运不发挥,每条能在原文找到出处,缺的节写「原帖未提及」;②轻校验:节头齐全+不得出现原文没有的数字/邮箱/URL;③原文永不覆盖,整理版存新列;④界面挂「AI 整理 · 原文在下」小注
批量只剩一个场景:要进「表格列/筛选」的字段(班次/语言)必须全量才有意义——后置,等 07-22 观察窗数据再定。与已有约定合拍:「公网链接到手后线上按需实时,不提前批量」(翻译已这么跑)。
